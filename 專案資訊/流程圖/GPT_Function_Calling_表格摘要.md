# GPT Function Calling 功能摘要表

## 功能總覽

| 函數名稱 | 類型 | 用途 | 參數 |
|---------|------|------|------|
| getTask | 查詢 | 獲取所有任務列表 | 無 |
| getTime | 查詢 | 獲取當前系統時間 | 無 |
| saveTask | 操作 | 新增單個或多個任務 | tasks 陣列 |
| deleteTask | 操作 | 刪除單個或多個任務 | taskIds 陣列 |
| updateTask | 操作 | 更新單個或多個任務 | tasks 陣列（含 taskId） |
| endConversation | 控制 | 正式結束對話 | 無 |

---

## 詳細規格表

### 1. getTask（獲取任務列表）

| 項目 | 說明 |
|------|------|
| **功能** | 從系統獲取使用者的所有任務資料 |
| **參數** | 無 |
| **回傳** | JSON 格式，包含當前時間與任務清單 |
| **使用情境** | "我現在有哪些任務？"、"幫我看看這週的安排" |
| **執行時機** | GPT 需要了解現有任務時自動呼叫 |

---

### 2. getTime（獲取當前時間）

| 項目 | 說明 |
|------|------|
| **功能** | 獲取當前日期與時間 |
| **參數** | 無 |
| **回傳** | 字串格式："yyyy/MM/dd HH:mm:ss" |
| **使用情境** | "現在幾點？"、"幫我安排今天下午的時間" |
| **執行時機** | 需要進行時間相關判斷或任務安排時 |

---

### 3. saveTask（儲存新任務）

| 項目 | 說明 |
|------|------|
| **功能** | 新增一個或多個任務到系統 |
| **必填參數** | • `title`: 任務標題<br>• `note`: 任務備註<br>• `category`: 任務分類<br>• `startDate`: 開始時間 (ISO 8601)<br>• `endDate`: 結束時間 (ISO 8601)<br>• `isAllDay`: "true" 或 "false"<br>• `isCompleted`: "true" 或 "false"<br>• `color`: 顏色代碼 |
| **顏色選項** | 紅色（緊急）、黃色（中等）、綠色（學習）、藍色（例行） |
| **使用情境** | "幫我安排明天早上9點讀英文" |
| **特色** | 支援批量新增，一次可新增多個任務 |

---

### 4. deleteTask（刪除任務）

| 項目 | 說明 |
|------|------|
| **功能** | 根據 ID 刪除一個或多個任務 |
| **必填參數** | • `taskIds`: 任務 ID 字串陣列 |
| **使用情境** | "刪除明天的英文課"、"把已完成的任務都刪掉" |
| **執行流程** | 1. 先用 getTask 獲取任務列表<br>2. 找出對應任務的 ID<br>3. 執行刪除操作 |
| **特色** | 支援批量刪除 |

---

### 5. updateTask（更新任務）

| 項目 | 說明 |
|------|------|
| **功能** | 修改現有任務的資訊 |
| **必填參數** | • `taskId`: 要更新的任務 ID |
| **選填參數** | title, note, category, startDate, endDate, isAllDay, isCompleted, color（只更新提供的欄位） |
| **使用情境** | "把明天的英文課改到下午3點"、"標記數學作業為已完成" |
| **更新策略** | • 只更新提供的欄位<br>• 未提供的欄位保持原值<br>• 支援部分更新 |
| **特色** | 支援批量更新 |

---

### 6. endConversation（結束對話）

| 項目 | 說明 |
|------|------|
| **功能** | 正式結束當前對話 |
| **參數** | 無 |
| **回傳** | 道別訊息 |
| **使用情境** | "謝謝你的幫忙"、"沒有其他事了" |
| **觸發條件** | • 使用者明確要結束對話<br>• 需求已完整處理<br>• 對話沒有明確目標 |
| **重要性** | **必須**呼叫此函數，不能只用文字說再見 |

---

## 函數呼叫模式

### 對話策略（tool_choice）

| 階段 | 設定值 | 說明 |
|------|--------|------|
| 第1次對話 | `none` | 先理解使用者需求，不呼叫函數 |
| 第2次對話 | `required` | 強制呼叫函數獲取資訊 |
| 後續對話 | `auto` 或 `required` | 根據上次回應類型動態調整 |

### 常見函數呼叫組合

| 使用者意圖 | 呼叫順序 | 說明 |
|-----------|----------|------|
| 查詢任務 | `getTask` | 單獨呼叫 |
| 新增任務 | `getTime` → `getTask` → `saveTask` | 確認時間與避免衝突 |
| 刪除任務 | `getTask` → `deleteTask` | 先查找任務 ID |
| 更新任務 | `getTask` → `updateTask` | 先查找任務 ID |
| 結束對話 | `endConversation` | 單獨呼叫 |

## 批量操作範例

### saveTask 批量新增

```json
{
  "tasks": [
    {
      "title": "讀英文",
      "startDate": "2025-01-16T09:00:00+08:00",
      "endDate": "2025-01-16T11:00:00+08:00",
      ...
    },
    {
      "title": "讀數學",
      "startDate": "2025-01-16T14:00:00+08:00",
      "endDate": "2025-01-16T16:00:00+08:00",
      ...
    }
  ]
}
```

### deleteTask 批量刪除

```json
{
  "taskIds": ["task1", "task2", "task3"]
}
```

### updateTask 批量更新

```json
{
  "tasks": [
    {
      "taskId": "task1",
      "isCompleted": "true"
    },
    {
      "taskId": "task2",
      "startDate": "2025-01-17T10:00:00+08:00"
    }
  ]
}
```

## 技術特點

| 特點 | 說明 |
|------|------|
| **批量操作** | saveTask, deleteTask, updateTask 均支援一次處理多個項目 |
| **錯誤處理** | 完整的參數驗證與錯誤訊息 |
| **資料同步** | 操作完成後自動重新整理任務清單與統計資料 |
| **安全性** | 透過 Firebase Cloud Functions 執行，確保使用者資料隔離 |
| **效能優化** | 本地快取機制，減少網路請求 |

## 實際應用場景

### 場景 1：規劃讀書計畫
```
使用者: "幫我安排這週每天下午2-4點讀書"
系統流程:
1. getTime() - 確認當前時間
2. getTask() - 檢查現有任務
3. saveTask() - 批量新增7個任務
4. endConversation() - 完成後結束對話
```

### 場景 2：調整任務時間
```
使用者: "把明天的英文課往後延一小時"
系統流程:
1. getTask() - 獲取所有任務
2. 找到「英文課」任務 ID
3. updateTask() - 更新開始/結束時間
4. endConversation() - 完成後結束對話
```

### 場景 3：清理已完成任務
```
使用者: "把今天已經完成的任務都刪掉"
系統流程:
1. getTask() - 獲取所有任務
2. 篩選出今天且 isCompleted=true 的任務
3. deleteTask() - 批量刪除
4. endConversation() - 完成後結束對話
```
