# GPT Function Calling 功能說明

本文件說明讀書助手 App 中 GPT 助理所使用的六個核心函數功能。

---

## 1. getTask - 獲取任務列表

### 功能
從系統中獲取使用者的所有任務資料，包含任務標題、時間、分類、完成狀態等資訊。

### 行為模式
- **參數**: 無需任何參數
- **回傳格式**: JSON 格式，包含當前時間與所有任務清單
- **資料來源**: 優先從本地快取讀取，若為空則向 Firebase 請求

### 使用例子
**使用者提問**:
> "我現在有哪些任務？"
> "幫我看看這週的安排"

**系統行為**:
1. GPT 呼叫 `getTask` 函數
2. 系統回傳所有任務的 JSON 資料
3. GPT 分析資料後以自然語言回覆使用者

**回傳資料範例**:
```json
{
  "currentTime": "2025-01-15T14:30:00.000+08:00",
  "existingTasks": [
    {
      "taskId": "abc123",
      "title": "準備期末考",
      "category": "學習",
      "startDate": "2025-01-20T09:00:00.000+08:00",
      "endDate": "2025-01-20T11:00:00.000+08:00",
      "isCompleted": false,
      "isAllDay": false,
      "note": "複習第1-5章"
    }
  ]
}
```

## 2. getTime - 獲取當前時間

### 功能
從系統獲取當前的日期與時間，用於時間相關的判斷與任務安排。

### 行為模式
- **參數**: 無需任何參數
- **回傳格式**: 字串，格式為 `yyyy/MM/dd HH:mm:ss`
- **時區**: 使用系統當前時區

### 使用例子
**使用者提問**:
> "現在幾點？"
> "幫我安排今天下午的讀書時間"

**系統行為**:
1. GPT 呼叫 `getTime` 函數
2. 系統回傳當前時間字串
3. GPT 根據時間進行後續規劃

**回傳資料範例**:
```
現在時間是：2025/01/15 14:30:45
```

---

## 3. saveTask - 儲存新任務

### 功能
將一個或多個新任務儲存到系統中，支援批量新增以提高效率。

### 行為模式
- **參數結構**:
  - `tasks`: 任務陣列（必填）
  - 每個任務必須包含以下欄位：
    - `title`: 任務標題（必填）
    - `note`: 任務備註（必填）
    - `category`: 任務分類（必填，系統會自動依內容判斷）
    - `startDate`: 開始時間（必填，ISO 8601 格式）
    - `endDate`: 結束時間（必填，ISO 8601 格式）
    - `isAllDay`: 是否全天任務（必填，"true" 或 "false"）
    - `isCompleted`: 是否已完成（必填，"true" 或 "false"）
    - `color`: 任務顏色（必填，四個預設顏色之一）

- **顏色選項**（根據任務重要性選擇）:
  - `0.7,0.16,0.13,0.4` (紅色) - 緊急或重要任務
  - `0.7,0.56,0,0.4` (黃色) - 中等優先級任務
  - `0.18,0.7,0.31,0.4` (綠色) - 學習或成長型任務
  - `0.29,0.28,0.7,0.4` (藍色) - 例行或常規任務

### 使用例子
**使用者提問**:
> "幫我安排明天早上9點到11點讀英文"
> "這週每天下午2點到4點都要複習數學"

**系統行為**:
1. GPT 先呼叫 `getTime` 確認當前時間
2. GPT 呼叫 `getTask` 確認現有任務，避免時間衝突
3. GPT 根據使用者需求產生任務參數
4. 呼叫 `saveTask` 函數儲存任務
5. 系統儲存後回覆成功訊息

**函數參數範例**:
```json
{
  "tasks": [
    {
      "title": "讀英文",
      "note": "複習英文單字與文法",
      "category": "學習",
      "startDate": "2025-01-16T09:00:00+08:00",
      "endDate": "2025-01-16T11:00:00+08:00",
      "isAllDay": "false",
      "isCompleted": "false",
      "color": "0.18,0.7,0.31,0.4"
    }
  ]
}
```

## 4. deleteTask - 刪除任務

### 功能
根據任務 ID 刪除一個或多個任務，支援批量刪除操作。

### 行為模式
- **參數結構**:
  - `taskIds`: 任務 ID 陣列（必填）
- **刪除邏輯**:
  - 只刪除符合 ID 的任務
  - 若 ID 不存在，系統會忽略該 ID
- **確認機制**: 刪除前會向使用者確認

### 使用例子
**使用者提問**:
> "刪除明天的英文課"
> "把今天已完成的任務都刪掉"

**系統行為**:
1. GPT 先呼叫 `getTask` 獲取任務列表
2. GPT 根據使用者描述找到對應的任務 ID
3. 呼叫 `deleteTask` 函數刪除任務
4. 系統回覆刪除結果

**函數參數範例**:
```json
{
  "taskIds": ["abc123", "def456"]
}
```

**系統回覆**:
```
已成功刪除 2 個任務
```

## 5. updateTask - 更新任務

### 功能
修改一個或多個現有任務的資訊，可以更新任務的任何欄位。

### 行為模式
- **參數結構**:
  - `tasks`: 任務更新陣列（必填）
  - 每個任務必須包含：
    - `taskId`: 要更新的任務 ID（必填）
    - 其他欄位（選填）：`title`, `note`, `category`, `startDate`, `endDate`, `isAllDay`, `isCompleted`, `color`

- **更新邏輯**:
  - 只有提供的欄位會被更新
  - 未提供的欄位保持原值
  - 標題或備註會根據需要適當更新

### 使用例子
**使用者提問**:
> "把明天的英文課改到下午3點"
> "標記數學作業為已完成"

**系統行為**:
1. GPT 先呼叫 `getTask` 獲取任務列表
2. GPT 找到對應的任務及其 ID
3. 根據使用者需求產生更新參數
4. 呼叫 `updateTask` 函數更新任務
5. 系統回覆更新結果

**函數參數範例（修改時間）**:
```json
{
  "tasks": [
    {
      "taskId": "abc123",
      "startDate": "2025-01-16T15:00:00+08:00",
      "endDate": "2025-01-16T17:00:00+08:00"
    }
  ]
}
```

**函數參數範例（標記完成）**:
```json
{
  "tasks": [
    {
      "taskId": "def456",
      "isCompleted": "true"
    }
  ]
}
```

## 6. endConversation - 結束對話

### 功能
正式結束當前對話，觸發系統的對話結束流程。

### 行為模式
- **參數**: 無需任何參數
- **觸發時機**:
  - 使用者明確表示要結束對話
  - 使用者的需求已完整處理完畢
  - 對話已經沒有明確目標或進展
- **重要**: GPT 必須呼叫此函數來結束對話，不能只用文字說再見

### 使用例子
**使用者提問**:
> "好的，謝謝你的幫忙"
> "沒有其他事了，bye"
> "就這樣吧"

**系統行為**:
1. GPT 判斷對話應該結束
2. 呼叫 `endConversation` 函數
3. 系統回傳道別訊息
4. 觸發對話結束信號，UI 可能會有相應變化

**系統回覆**:
```
感謝您的使用，祝您學習順利！如果需要再次規劃，隨時歡迎找我聊天。
```

## 函數呼叫策略

根據 ChatViewModel.swift 第 1118-1132 行的實作，系統使用智慧的 `tool_choice` 策略：

1. **第一次對話**（sendToGPTCount = 0）: `tool_choice = "none"`
   - 讓 GPT 先理解使用者需求，不呼叫函數

2. **第二次對話**（sendToGPTCount = 1）: `tool_choice = "required"`
   - 強制 GPT 呼叫至少一個函數來獲取資訊

3. **後續對話**: 根據上一次的行為動態調整
   - 若上次是文字回應 → 下次設為 "required"（鼓勵使用函數）
   - 若上次是函數呼叫 → 下次設為 "auto"（讓 GPT 自行判斷）

這種策略確保對話既能獲取必要資訊，又能保持自然流暢。

## 技術實作重點

### 批量操作支援
`saveTask`, `deleteTask`, `updateTask` 都支援批量操作，可以一次處理多個任務，大幅提升效率。

### 錯誤處理
所有函數都包含完善的錯誤處理機制：
- 參數驗證
- 資料格式檢查
- 網路錯誤重試
- 使用者友善的錯誤訊息

### 資料同步
- 任務操作完成後會自動重新整理任務清單
- 統計資料會即時更新
- 確保 UI 與資料庫保持同步

### 安全性考量
- 所有操作都需要使用者登入
- 任務只能存取當前使用者的資料
- 透過 Firebase Cloud Functions 執行，確保資料安全
