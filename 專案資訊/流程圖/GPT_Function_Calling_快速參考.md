# GPT Function Calling 快速參考卡

## 六大核心函數

### 📋 查詢類

#### 1. getTask
- **用途**: 獲取所有任務
- **參數**: 無
- **回傳**: JSON 任務清單
- **例句**: "我現在有哪些任務？"

#### 2. getTime
- **用途**: 獲取當前時間
- **參數**: 無
- **回傳**: 時間字串
- **例句**: "現在幾點？"

---

### ⚙️ 操作類

#### 3. saveTask
- **用途**: 新增任務（支援批量）
- **參數**:
  ```
  tasks: [
    {
      title, note, category,
      startDate, endDate,
      isAllDay, isCompleted, color
    }
  ]
  ```
- **例句**: "幫我安排明天早上9點讀英文"

#### 4. deleteTask
- **用途**: 刪除任務（支援批量）
- **參數**: `taskIds: ["id1", "id2"]`
- **例句**: "刪除明天的英文課"

#### 5. updateTask
- **用途**: 更新任務（支援批量）
- **參數**:
  ```
  tasks: [
    {
      taskId: "必填",
      其他欄位: "選填"
    }
  ]
  ```
- **例句**: "把英文課改到下午3點"

---

### 🎯 控制類

#### 6. endConversation
- **用途**: 結束對話
- **參數**: 無
- **觸發**: 使用者道別或需求完成
- **例句**: "謝謝，沒有其他事了"

---

## 函數呼叫流程

```
使用者提問
    ↓
GPT 判斷意圖
    ↓
[第1次: none] → 理解需求
[第2次: required] → 強制呼叫函數
[後續: auto/required] → 智慧選擇
    ↓
執行函數並回傳結果
    ↓
GPT 整合並回覆
```

---

## 常見組合模式

| 情境 | 函數呼叫順序 |
|------|-------------|
| 新增任務 | getTime → getTask → saveTask |
| 刪除任務 | getTask → deleteTask |
| 更新任務 | getTask → updateTask |
| 查詢任務 | getTask |
| 結束對話 | endConversation |

| 顏色 | 代碼 | 用途 |
|------|------|------|
| 🔴 紅色 | `0.7,0.16,0.13,0.4` | 緊急/重要 |
| 🟡 黃色 | `0.7,0.56,0,0.4` | 中等優先 |
| 🟢 綠色 | `0.18,0.7,0.31,0.4` | 學習/成長 |
| 🔵 藍色 | `0.29,0.28,0.7,0.4` | 例行/常規 |

## 日期時間格式

- **輸入格式**: ISO 8601
  - 範例: `"2025-01-16T09:00:00+08:00"`
- **回傳格式**: `yyyy/MM/dd HH:mm:ss`
  - 範例: `"2025/01/16 09:00:00"`

---

## 批量操作限制

- **saveTask**: 建議一次最多 100 個任務
- **deleteTask**: 無限制
- **updateTask**: 無限制

---

## 錯誤處理

所有函數都包含：
- ✅ 參數驗證
- ✅ 格式檢查
- ✅ 網路重試（最多3次）
- ✅ 友善錯誤訊息

## 使用範例（實際對話）

### 範例 1: 新增單個任務
```
使用者: "幫我安排明天下午2點到4點讀數學"

GPT 執行:
1. getTime() → "2025/01/15 10:30:00"
2. getTask() → 檢查明天是否有衝突
3. saveTask({
     title: "讀數學",
     startDate: "2025-01-16T14:00:00+08:00",
     endDate: "2025-01-16T16:00:00+08:00",
     ...
   })

回覆: "好的，我已經幫你安排明天下午2點到4點讀數學。"
```

### 範例 2: 批量新增任務
```
使用者: "這週每天下午2點都要讀書，每次2小時"

GPT 執行:
1. getTime() → 確認今天日期
2. getTask() → 檢查是否有衝突
3. saveTask({
     tasks: [
       { title: "讀書", startDate: "週一...", ... },
       { title: "讀書", startDate: "週二...", ... },
       ...共7個任務
     ]
   })

回覆: "已幫你安排好本週每天下午2-4點的讀書時間。"
```

### 範例 3: 修改任務
```
使用者: "把明天的數學課往後延1小時"

GPT 執行:
1. getTask() → 找到"數學課"任務 (id: "abc123")
2. updateTask({
     tasks: [{
       taskId: "abc123",
       startDate: "2025-01-16T15:00:00+08:00",
       endDate: "2025-01-16T17:00:00+08:00"
     }]
   })

回覆: "已將明天的數學課時間調整為下午3點到5點。"
```

### 範例 4: 刪除任務
```
使用者: "刪除今天所有已完成的任務"

GPT 執行:
1. getTask() → 獲取所有任務
2. 篩選: 今天 + isCompleted=true
   找到: ["task1", "task3", "task5"]
3. deleteTask({ taskIds: ["task1", "task3", "task5"] })

回覆: "已刪除今天的3個已完成任務。"
```

4. **資料同步**: 操作後立即更新 UI 與本地快取
5. **安全隔離**: 透過 Cloud Functions 確保使用者資料安全

- **錯誤重試次數**: 最多 3 次
- **重試間隔**: 1.5 秒 × 嘗試次數
