 步驟 1：安裝基礎工具
────────────────────────────────────────────────────────────────────────
1. 安裝 Xcode (從 App Store)

   - 開啟 App Store 搜尋 "Xcode"
   - 安裝最新版本 (約 15GB)
   - 安裝完成後開啟 Xcode 同意授權條款

   驗證安裝：
   ```bash
   xcodebuild -version
   ```

2. 安裝 Xcode Command Line Tools

   ```bash
   xcode-select --install
   ```

3. 驗證 Swift 版本

   ```bash
   swift --version
   ```

   應顯示：Apple Swift version 6.1.2 或更新

4. 安裝 Homebrew (套件管理器)

   ```bash
   /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
   ```

5. 安裝 Node.js (使用 Homebrew)

   ```bash
   brew install node@20
   ```

   驗證安裝：
   ```bash
   node --version   # 應顯示 v20.x.x
   npm --version    # 應顯示 10.x.x
   ```

6. 安裝 Firebase CLI

   ```bash
   npm install -g firebase-tools
   ```

   驗證安裝：
   ```bash
   firebase --version
   ```

7. 安裝 Git (通常 macOS 已內建)

   ```bash
   git --version
   ```
────────────────────────────────────────────────────────────────────────


 步驟 2：Clone 專案
────────────────────────────────────────────────────────────────────────
1. 使用 Git Clone 專案

   ```bash
   cd ~/Developer  # 或您的專案目錄
   git clone <repository-url> studyAssistant
   cd studyAssistant
   ```

2. 檢查專案結構

   ```bash
   ls -la
   ```

   應該看到：
   - studyAssistant/           (iOS 程式碼)
   - firebase_cloud_function/  (Cloud Functions)
   - studyAssistant.xcodeproj/ (Xcode 專案)
   - 專案資訊/                  (文檔)
────────────────────────────────────────────────────────────────────────


 步驟 3：設定 iOS 專案
────────────────────────────────────────────────────────────────────────
1. 開啟 Xcode 專案

   ```bash
   open studyAssistant.xcodeproj
   ```

   或雙擊 studyAssistant.xcodeproj 檔案

2. 設定開發團隊 (Development Team)

   - 在 Xcode 中選擇專案根目錄
   - 選擇 Target: studyAssistant
   - 切換到 "Signing & Capabilities" 頁籤
   - 選擇您的 Apple ID 開發團隊
   - Bundle Identifier 建議修改為您自己的 (例如: com.yourname.studyAssistant)

3. 安裝 Swift Package Dependencies

   Xcode 應該會自動下載依賴，如果沒有：

   - File → Packages → Resolve Package Versions
   - 等待下載完成 (Firebase SDK 約 100MB)

4. 配置 Firebase

    重要：您需要自己的 Firebase 專案

   a. 前往 https://console.firebase.google.com/
   b. 建立新專案或使用現有專案
   c. 新增 iOS 應用程式
   d. 下載 GoogleService-Info.plist
   e. 將檔案拖入 Xcode 專案根目錄 (確認 "Copy items if needed")

    GoogleService-Info.plist 不應提交到 Git (已在 .gitignore 中)

5. 配置 Google Sign-In

   a. 在 Firebase Console 啟用 Google 驗證
   b. 從 GoogleService-Info.plist 複製 REVERSED_CLIENT_ID
   c. 在 Xcode 中：
      - 選擇 Target → Info
      - 展開 "URL Types"
      - 確認有一個 URL Scheme 為您的 REVERSED_CLIENT_ID

6. 建立並執行

   - 選擇模擬器 (建議: iPhone 15 Pro, iOS 18.0)
   - 按下 Cmd+R 或點擊 "Play" 按鈕
   - 首次建置可能需要 5-10 分鐘

 步驟 4：設定 Cloud Functions
────────────────────────────────────────────────────────────────────────
1. 進入 Cloud Functions 目錄

   ```bash
   cd firebase_cloud_function/database_functions
   ```

2. 安裝 Node.js 依賴

   ```bash
   npm install
   ```

   這會安裝 package.json 中列出的所有依賴 (~150MB)

3. 登入 Firebase

   ```bash
   firebase login
   ```

   會開啟瀏覽器進行 Google 帳號驗證

4. 設定 Firebase 專案

   ```bash
   firebase use --add
   ```

   選擇您在步驟 3-4 建立的 Firebase 專案

5. 編譯 TypeScript

   ```bash
   npm run build
   ```

   這會將 src/ 中的 TypeScript 編譯到 lib/

6. 測試本地 Functions (可選)

   ```bash
   npm run serve
   ```

   啟動 Firebase Emulator，可在本地測試 Functions

7. 部署到 Firebase

   ```bash
   npm run deploy
   ```

   或使用完整命令：
   ```bash
   firebase deploy --only functions
   ```

   首次部署可能需要 5-10 分鐘
   部署區域：asia-east1

8. 驗證部署

   前往 Firebase Console → Functions
   應該看到 8 個已部署的函數：

   - fetchTasks

   - updateTaskInstance

                           三、專案開發工作流程

 日常開發流程
────────────────────────────────────────────────────────────────────────
1. 啟動開發

   ```bash
   cd ~/Developer/studyAssistant
   git pull origin main           # 拉取最新程式碼
   git checkout -b feature/新功能  # 建立功能分支
   open studyAssistant.xcodeproj  # 開啟 Xcode
   ```

2. 進行開發

   - 在 Xcode 中編輯程式碼
   - 遵循 MVVM 架構原則 (參考 02_架構設計.txt)
   - 定期儲存並建置 (Cmd+B)

3. 測試變更

   - 在模擬器測試 (Cmd+R)
   - 在真機測試 (連接 iPhone)
   - 執行單元測試 (Cmd+U)

4. 提交變更

   ```bash
   git add .
   git commit -m "feat: 簡短描述變更內容"
   git push origin feature/新功能
   ```

5. 建立 Pull Request

   - 前往 GitHub/GitLab
   - 從 feature/新功能 → main
   - 填寫 PR 描述
   - 等待 Code Review

6. 合併並部署

   ```bash
   git checkout main
   git pull origin main

   # 如果有 Cloud Functions 變更
   cd firebase_cloud_function/database_functions
   npm run build
   npm run deploy
   ```

 程式碼風格指南
────────────────────────────────────────────────────────────────────────
Swift 程式碼風格：

1. 命名規則
   - 類別/結構體：PascalCase (TodoViewModel, UserProfile)
   - 變數/函數：camelCase (fetchTasks, isLoading)
   - 常數：camelCase (maxRetryCount)
   - 列舉：PascalCase，成員 camelCase (RepeatType.daily)

2. 縮排
   - 使用 4 個空格 (不使用 Tab)
   - Xcode 預設設定

3. 程式碼組織
   - 使用 // MARK: - 分隔程式碼區塊
   - 順序：屬性 → 初始化 → 生命週期 → 公開方法 → 私有方法

   範例：
   ```swift
   class TodoViewModel: ObservableObject {
       // MARK: - Properties
       @Published var tasks: [TodoTask] = []

       private let firebaseService: FirebaseService

       // MARK: - Initialization
       init(firebaseService: FirebaseService = .shared) {
           self.firebaseService = firebaseService
       }

       // MARK: - Public Methods
       func addTask(_ task: TodoTask) async {
           // ...
       }

       // MARK: - Private Methods
       private func handleError(_ error: Error) {
           // ...
       }
   }
   ```

4. SwiftUI 視圖組織
   - 將複雜視圖拆分為子視圖
   - 使用 ViewBuilder 組織邏輯
   - 提取常數到頂部

   範例：
   ```swift
   struct TodoView: View {
       // MARK: - Properties
       @StateObject private var viewModel = TodoViewModel()

       // MARK: - Constants
       private let spacing: CGFloat = 12

       // MARK: - Body
       var body: some View {
           NavigationView {
               content
           }
       }

       // MARK: - Subviews
       private var content: some View {
           List {
               ForEach(viewModel.tasks) { task in
                   TaskRow(task: task)
               }
           }
       }
   }
   ```

5. 非同步處理
   - 使用 async/await 而非閉包
   - ViewModel 標記為 @MainActor
   - 在 View 中使用 Task {}

   範例：
   ```swift
   // ViewModel
   @MainActor
   class TodoViewModel: ObservableObject {
       func addTask(_ task: TodoTask) async {
           // 自動在 Main Thread
       }
   }

   // View
   Button("新增") {
       Task {
           await viewModel.addTask(newTask)
       }
   }
   ```

TypeScript/JavaScript 程式碼風格：

1. 命名規則
   - 函數：camelCase (createTask, validateData)
   - 介面/型別：PascalCase (TodoTask, UserProfile)
   - 常數：UPPER_SNAKE_CASE (MAX_RETRY_COUNT)

2. 縮排
   - 使用 2 個空格 (遵循 Google Style)

3. ESLint 規則
   - 使用專案的 .eslintrc.js
   - 執行檢查：npm run lint
   - 自動修正：eslint --fix src/

4. TypeScript 型別
   - 明確標記所有參數和回傳值型別
   - 使用 interface 而非 type (除非必要)
   - 避免使用 any

   範例：
   ```typescript
   interface CreateTaskRequest {
     task: TodoTask;
   }

   export const createTask = onCall(
     { region: 'asia-east1' },
     async (request: CallableRequest<CreateTaskRequest>): Promise<Response> => {
       // ...
     }
   );
   ```

 任務 4：新增 Cloud Function API
────────────────────────────────────────────────────────────────────────
1. 在 firebase_cloud_function/database_functions/src/index.ts 新增函數

   ```typescript
   import { onCall, CallableRequest } from 'firebase-functions/v2/https';
   import { getFirestore } from 'firebase-admin/firestore';

   interface CreateNotificationRequest {
     notification: {
       userId: string;
       title: string;
       body: string;
       scheduledFor: string; // ISO timestamp
     };
   }

   interface Response {
     success: boolean;
     notificationId?: string;
     error?: string;
   }

   export const createNotification = onCall(
     { region: 'asia-east1' },
     async (request: CallableRequest<CreateNotificationRequest>): Promise<Response> => {
       try {
         const { notification } = request.data;

         // 驗證輸入
         if (!notification.userId || !notification.title) {
           return { success: false, error: '缺少必要欄位' };
         }

         // 儲存到 Firestore
         const db = getFirestore();
         const notificationRef = await db.collection('notifications').add({
           ...notification,
           isRead: false,
           createdAt: new Date().toISOString(),
         });

         return {
           success: true,
           notificationId: notificationRef.id,
         };
       } catch (error) {
         console.error('Error creating notification:', error);
         return { success: false, error: String(error) };
       }
     }
   );
   ```

2. 編譯並部署

   ```bash
   cd firebase_cloud_function/database_functions
   npm run build
   npm run deploy
   ```

3. 在 iOS 端呼叫 (在 FirebaseService.swift 中)

   ```swift
   func createNotification(_ notification: Notification) async throws -> String {
       let functions = Functions.functions(region: "asia-east1")

       let notificationData: [String: Any] = [
           "userId": notification.userId,
           "title": notification.title,
           "body": notification.body,
           "scheduledFor": ISO8601DateFormatter().string(from: notification.scheduledFor)
       ]

       let result = try await functions.httpsCallable("createNotification").call([
           "notification": notificationData
       ])

       guard let data = result.data as? [String: Any],
             let success = data["success"] as? Bool,
             success,
             let notificationId = data["notificationId"] as? String else {
           throw DataServiceError.networkError
       }

       return notificationId
   }
   ```

   ```typescript
   export const migrateTasksV2 = onRequest(
     { region: 'asia-east1' },
     async (req, res) => {
       const db = getFirestore();
       const tasksSnapshot = await db.collection('tasks').get();

       const batch = db.batch();
       tasksSnapshot.docs.forEach((doc) => {
         const taskData = doc.data();
         // 新增新欄位
         batch.update(doc.ref, {
           priority: taskData.priority || 'medium',
           version: (taskData.version || 0) + 1,
         });
       });

       await batch.commit();
       res.send({ success: true, migrated: tasksSnapshot.size });
     }
   );
   ```


3. 排程本地通知

   ```swift
   func scheduleNotification(for task: TodoTask) {
       let content = UNMutableNotificationContent()
       content.title = "任務提醒"
       content.body = task.title
       content.sound = .default

       let triggerDate = Calendar.current.dateComponents(
           [.year, .month, .day, .hour, .minute],
           from: task.startDate
       )
       let trigger = UNCalendarNotificationTrigger(dateMatching: triggerDate, repeats: false)

       let request = UNNotificationRequest(
           identifier: task.id,
           content: content,
           trigger: trigger
       )

       UNUserNotificationCenter.current().add(request) { error in
           if let error = error {
               print("排程通知失敗: \(error)")
           }
       }
   }
   ```

4. 取消通知

   ```swift
   func cancelNotification(for taskId: String) {
       UNUserNotificationCenter.current().removePendingNotificationRequests(withIdentifiers: [taskId])
   }
   ```

❓ 常見問題與解決方案
────────────────────────────────────────────────────────────────────────
問題：Xcode 無法建置，出現 "Command PhaseScriptExecution failed"

解決方案：
  1. Clean Build Folder (Shift+Cmd+K)
  2. 刪除 DerivedData：
     ```bash
     rm -rf ~/Library/Developer/Xcode/DerivedData
     ```
  3. 重新開啟 Xcode
────────────────────────────────────────────────────────────────────────

問題：Swift Package 下載失敗

解決方案：
  1. File → Packages → Reset Package Caches
  2. File → Packages → Update to Latest Package Versions
  3. 檢查網路連線
  4. 嘗試使用 VPN (某些地區可能無法存取 GitHub)
────────────────────────────────────────────────────────────────────────

問題：Google Sign-In 失敗，顯示 "Error 10"

解決方案：
  1. 檢查 GoogleService-Info.plist 是否正確加入專案
  2. 驗證 REVERSED_CLIENT_ID 是否在 URL Types 中
  3. 檢查 Bundle Identifier 是否與 Firebase Console 一致
  4. 確認在 Firebase Console 啟用了 Google 驗證
────────────────────────────────────────────────────────────────────────

問題：Firebase Functions 部署失敗

解決方案：
  1. 檢查 Node.js 版本 (必須是 20)
     ```bash
     node --version
     ```
  2. 清除 node_modules 重新安裝
     ```bash
     rm -rf node_modules package-lock.json
     npm install
     ```
  3. 檢查 Firebase 專案配額 (免費方案有限制)
  4. 查看詳細錯誤訊息：
     ```bash
     firebase deploy --only functions --debug
     ```
────────────────────────────────────────────────────────────────────────

問題：Firestore 資料無法同步

解決方案：
  1. 檢查網路連線
  2. 驗證使用者已登入 (userId 不為 nil)
  3. 檢查 Firestore Security Rules
  4. 在 Firebase Console 手動新增資料測試
  5. 檢查 FirebaseService 的錯誤日誌
────────────────────────────────────────────────────────────────────────

問題：App 在真機上閃退，但模擬器正常

解決方案：
  1. 連接真機，在 Xcode 中查看 Console 日誌
  2. 檢查是否有權限問題 (相機、位置、通知等)
  3. 驗證 Provisioning Profile 正確
  4. 檢查是否有使用模擬器限定的 API
────────────────────────────────────────────────────────────────────────

問題：SwiftUI Preview 無法顯示

解決方案：
  1. 確保 Preview 程式碼正確
  2. Cmd+Option+P 重新整理 Preview
  3. Editor → Canvas (Cmd+Option+Enter) 顯示 Canvas
  4. 重啟 Xcode
  5. Clean Build Folder
────────────────────────────────────────────────────────────────────────

問題：計時器在背景不準確

解決方案：
  - iOS 會暫停背景 App 的 Timer
  - 使用時間戳記而非 Timer 計數
  - 記錄開始時間，回到前景時計算經過時間

  範例：
  ```swift
  @Published var elapsedTime: TimeInterval = 0
  private var startTime: Date?

  func startTimer() {
      startTime = Date()
      timer = Timer.scheduledTimer(withTimeInterval: 1, repeats: true) { _ in
          if let startTime = self.startTime {
              self.elapsedTime = Date().timeIntervalSince(startTime)
          }
      }
  }
  ```
────────────────────────────────────────────────────────────────────────

問題：ChatGPT 回應太慢或超時

解決方案：
  1. 檢查 OpenAI API 配額
  2. 使用較小的模型 (gpt-4o-mini 而非 gpt-4)
  3. 減少 max_tokens 參數
  4. 實作請求超時處理：
     ```swift
     let task = Task {
         try await chatViewModel.sendMessage(message)
     }

     try await withTimeout(seconds: 30) {
         try await task.value
     }
     ```

                        八、效能最佳化建議
================================================================================

 iOS App 效能優化
────────────────────────────────────────────────────────────────────────
1. SwiftUI 視圖優化

   - 使用 @State 而非 @Published (對於純 View 內的狀態)
   - 避免在 body 中執行複雜計算
   - 使用 .id() 強制重新渲染而非條件判斷
   - 拆分大型視圖為小元件

    不佳：
   ```swift
   var body: some View {
       List {
           ForEach(viewModel.tasks.filter { !$0.isCompleted }) { task in
               // ...
           }
       }
   }
   ```

    較佳：
   ```swift
   var incompleteTasks: [TodoTask] {
       viewModel.tasks.filter { !$0.isCompleted }
   }

   var body: some View {
       List {
           ForEach(incompleteTasks) { task in
               // ...
           }
       }
   }
   ```

2. 減少不必要的 @Published 更新

   - 只在值真的改變時才更新
   - 使用 didSet 檢查

   ```swift
   @Published var tasks: [TodoTask] = [] {
       didSet {
           guard tasks != oldValue else { return }
           // 實際有變更才執行
       }
   }
   ```

3. 非同步圖片載入

   - 使用 AsyncImage 或 Kingfisher
   - 避免在主執行緒載入大圖

4. 列表效能優化

   - 使用 LazyVStack 而非 VStack (大量項目時)
   - 限制每次載入的資料量 (分頁載入)

5. 記憶體管理

   - 使用 [weak self] 避免循環參照
   - 取消不需要的 Combine 訂閱

   ```swift
   Timer.publish(every: 1, on: .main, in: .common)
       .autoconnect()
       .sink { [weak self] _ in
           self?.updateTime()
       }
       .store(in: &cancellables)
   ```
────────────────────────────────────────────────────────────────────────


 Firebase 效能優化
────────────────────────────────────────────────────────────────────────
1. Firestore 查詢優化

   - 使用索引 (Firebase 會自動建議)
   - 限制查詢結果數量

   ```swift
   // 限制 20 筆
   db.collection("tasks")
       .whereField("userId", isEqualTo: userId)
       .order(by: "createdAt", descending: true)
       .limit(to: 20)
   ```

2. 批次寫入

   - 多個寫入操作使用 batch

   ```swift
   let batch = db.batch()
   tasks.forEach { task in
       let ref = db.collection("tasks").document(task.id)
       batch.setData(task.toDictionary(), forDocument: ref)
   }
   try await batch.commit()
   ```

3. 快取策略

   - 使用 Firestore 離線快取
   - 本地優先讀取，背景同步

   ```swift
   let settings = FirestoreSettings()
   settings.cacheSettings = PersistentCacheSettings()
   db.settings = settings
   ```

4. 減少 Cloud Functions 冷啟動

   - 使用 Min Instances (付費功能)
   - 優化函數大小 (減少依賴)
   - 使用 TypeScript 而非 JavaScript (更快)

5. 資料結構優化

   - 反正規化 (避免多次查詢)
   - 儲存計算結果而非每次計算
────────────────────────────────────────────────────────────────────────


 網路效能優化
────────────────────────────────────────────────────────────────────────
1. 減少網路請求次數

   - 批次操作
   - 使用 Firestore Realtime Listener 而非輪詢

2. 壓縮資料

   - 只傳輸必要欄位
   - 使用較短的欄位名稱

3. 錯誤處理與重試

   - 實作指數退避重試
   - 網路錯誤時顯示離線提示
