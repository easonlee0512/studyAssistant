                           版本命名規則
================================================================================

版本號格式: vX.Y.Z

  X (主版本號):
    - 重大架構變更
    - 不向下相容的 API 變更
    - 重要功能完全重寫

  Y (次版本號):
    - 新功能新增
    - 重要功能改進
    - 向下相容的 API 變更

  Z (修訂版本號):
    - Bug 修復
    - 小幅改進
    - 效能優化

標籤:
  -  新功能
  -  改進
  -  修復
  -  效能
  -  安全
  -  文檔
  -  移除

v1.0.0 - 正式版本 (2025-11-01)
────────────────────────────────────────────────────────────────────────

 重大里程碑: 專案正式發布

新增功能:
   完整的待辦事項管理系統
     - 支援每日/每週/每月重複任務
     - 任務分類與顏色標籤
     - 任務實例獨立完成狀態
     - 跨天任務支援

   番茄鐘計時器
     - 專注計時功能
     - 背景計時支援
     - 計時記錄儲存
     - 科目分類統計

   智慧日曆功能
     - 月曆視圖
     - 任務視覺化顯示
     - 跨天任務自動排列
     - 日期選擇與任務查看

   AI 聊天助手
     - OpenAI GPT-4o-mini 整合
     - 多聊天室管理
     - Markdown 渲染
     - 串流回應
     - Function Calling (新增/刪除/修改任務)

   日曆助手
     - 自動分析任務與學習狀態
     - 智慧安排學習計劃
     - 任務自動新增到日曆
     - 麥克風語音輸入

   學習統計
     - 每日專注時間統計
     - 科目分類統計
     - 任務完成率
     - 長條圖視覺化

   使用者系統
     - Google OAuth 2.0 登入
     - 使用者資料管理
     - 個人化設定
     - 鼓勵語句自訂

技術架構:
   MVVM 架構
     - 清晰的職責分離
     - SwiftUI + Combine
     - Protocol-Oriented Programming

   Firebase 整合
     - Firestore 資料庫
     - Firebase Authentication
     - Cloud Functions (24 個 API)
     - 資料同步與離線支援

   Swift Concurrency
     - async/await
     - Task-based concurrency
     - @MainActor 執行緒安全

程式碼規模:
  - iOS App: 38 個 Swift 檔案, ~745 KB
  - Cloud Functions: 約 2,000 行 TypeScript
  - 總計: 約 15,000 行程式碼

已知問題:
  - 跨天任務在日曆中的顯示疊層偶有不正確
  - 週月重複的跨天任務只顯示第一天
  - 下個月/上個月的任務偶爾不正常顯示

相容性:
  - iOS 17.0+
  - Swift 6.1.2+
  - Node.js 20

================================================================================
                         版本發布流程
================================================================================

1. 開發階段
   - 在 develop 分支開發新功能
   - 使用 feature/* 分支隔離功能開發
   - 定期合併到 develop

2. 測試階段
   - 內部測試 (Alpha)
   - 小範圍測試 (Beta)
   - TestFlight 發布

3. 發布準備
   - 更新版本號 (Xcode Project Settings)
   - 更新 CHANGELOG
   - 創建 Git Tag
   - 準備 App Store 素材

4. 正式發布
   - 合併到 main 分支
   - 創建 Release Tag
   - App Store 提交審核
   - 發布公告

5. 發布後
   - 監控錯誤報告
   - 收集使用者反饋
   - 規劃下一版本
