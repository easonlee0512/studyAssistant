===========================================
專案概述 (Project Overview)
===========================================

【專案名稱】
studyAssistant - 學習助手

【專案標語】
你的個人學習夥伴

===========================================
一、專案目標
===========================================

【主要目標】
打造一款智能化的學習管理應用，幫助學生有效規劃學習時間、追蹤學習進度，
並透過 AI 助手提供個性化的學習建議，提升學習效率和成就感。

【次要目標】
1. 提供簡潔直觀的使用者介面
2. 實現跨設備的資料同步
3. 利用 AI 技術提供智能學習建議
4. 幫助使用者養成良好的學習習慣
5. 提供詳細的學習統計與分析

【目標用戶】
- 國中生、高中生
- 大學生、研究生
- 語言學習者
- 自學者
- 任何需要管理學習進度的人

===========================================
二、核心功能簡介
===========================================

【1. 任務管理 (Todo Management)】
功能描述:
- 創建、編輯、刪除學習任務
- 支援重複任務（每天、每週、每月）
- 任務分類管理（科目分類）
- 任務優先級設定
- 子任務支援
- 任務完成追蹤

技術特點:
- 樂觀更新機制（即時 UI 反饋）
- Firestore 即時同步
- 服務端任務實例生成
- 衝突解決機制

【2. 專注計時器 (Focus Timer)】
功能描述:
- 正數計時器（從0開始）
- 倒數計時器（設定目標時間）
- 暫停/繼續功能
- 計時記錄保存
- 背景計時支援

技術特點:
- 時間差計算法（避免累積誤差）
- 生命週期管理（前景/背景切換）
- UserDefaults 本地持久化
- 通知提醒

【3. AI 聊天助手 (AI Chat Assistant)】
功能描述:
- 與 GPT 對話獲取學習建議
- 自動解析任務相關操作
- Function Calling 整合
- 串流回應顯示
- 對話歷史記錄

技術特點:
- OpenAI GPT-4 整合
- Function Calling 機制（6種函數）
- Token 使用追蹤
- 自動捲動到最新訊息
- 重試機制（指數退避算法）

【4. 統計分析 (Statistics & Analytics)】
功能描述:
- 按科目分類的學習統計
- 任務完成率追蹤
- 專注時長統計
- 每日/總體統計視圖
- 進度條視覺化

技術特點:
- 實時統計計算
- Firestore 聚合查詢
- 自動同步機制
- Token 使用統計

【5. 日曆助手 (Calendar Assistant)】
功能描述:
- AI 自動安排每日任務
- 每日學習計劃生成
- 批次任務操作
- 撤銷/重做功能
- 自動更新機制

技術特點:
- GPT Function Calling
- 16 條規則系統提示
- 每日自動更新檢查
- 樂觀更新與回滾
- Singleton 模式

【6. 用戶設定 (User Settings & Profile)】
功能描述:
- Google OAuth 登入
- 個人資料管理
- 學習偏好設定
- 應用設定（通知等）
- 目標日期追蹤

技術特點:
- Google OAuth 2.0
- Firebase Authentication
- Firestore 即時監聽
- 多層網路檢查
- 版本控制（樂觀並發）

三、技術棧總覽
===========================================

【前端技術】
語言與框架:
- Swift 5.9+
- SwiftUI (iOS 16.0+)
- Combine Framework

開發工具:
- Xcode 15+
- iOS Simulator
- SF Symbols

【後端技術】
雲端服務:
- Firebase
  ├─ Authentication (使用者認證)
  ├─ Firestore (NoSQL 資料庫)
  └─ Cloud Functions (伺服器端邏輯)

Cloud Functions:
- 語言: TypeScript
- 運行環境: Node.js 20
- 部署: Firebase CLI

【第三方服務】
AI 服務:
- OpenAI API
  ├─ GPT-4o-mini
  ├─ GPT-4o
  └─ Function Calling

認證服務:
- Google Sign-In SDK

【資料庫】
主要資料庫:
- Firestore (雲端 NoSQL)

本地儲存:
- UserDefaults (偏好設定、計時記錄)

【架構模式】
- MVVM (Model-View-ViewModel)
- Singleton Pattern
- Repository Pattern
- Observer Pattern
- Protocol-Oriented Programming

五、專案規模
===========================================

【代碼統計】
iOS App:
- Swift 檔案: 40+ files
- 總代碼行數: ~15,000+ lines
- Models: 10+ files
- ViewModels: 8+ files
- Views: 20+ files
- Services: 3+ files

Cloud Functions:
- TypeScript 檔案: 25+ files
- 總代碼行數: ~8,000+ lines
- Functions: 24+ functions
- Types: 完整型別定義
- Validators: Zod 驗證結構
- Error Handling: 集中式錯誤處理

【資料庫】
Firestore Collections:
- users: 用戶資訊
- profiles: 用戶個人檔案
- settings: 應用設定
- studySettings: 學習偏好設定
- tasks: 任務資料
- statistics: 統計資料
- timerRecords: 計時記錄
- chatHistories: 聊天記錄

【API 端點】
Cloud Functions: 24+ endpoints
- User Management: 4 functions
- Task Management: 8 functions
- Statistics: 4 functions
- Chat: 2 functions
- Calendar: 3 functions
- Settings: 3 functions
