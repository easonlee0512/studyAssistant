===========================================
計時器功能 (Timer / Pomodoro)
===========================================

【功能概述】
計時器功能提供正向計時和倒數計時兩種模式,支援可視化的圓環進度顯示、時間拖動調整、計時記錄保存、以及與任務和統計數據的深度整合。

【1. TimerRecord 結構】

計時記錄資料結構:

struct TimerRecord: Identifiable, Codable {
    var id: String              // UUID 字串
    var userId: String          // 用戶ID
    var subject: String         // 科目/主題
    var startTime: Date         // 開始時間
    var endTime: Date           // 結束時間
    var duration: Int           // 持續時間(秒)
    var isCompleted: Bool       // 是否完成
    var createdAt: Date         // 建立時間
    var lastModifiedAt: Date    // 最後修改時間
    var isDeleted: Bool         // 是否已刪除(軟刪除)
}

【計算屬性】
- durationInMinutes: Double
  └─ 將持續時間從秒轉換為分鐘

【初始化方法】

【時間差計算法原理】

傳統方法問題:
- 每秒減少或增加 timeRemaining
- 累積誤差,不精確
- 暫停/恢復後不準確

時間差計算法:
1. 記錄開始時間點: startTime
2. 記錄初始剩餘時間: initialTimeRemaining (倒數)
3. 每次更新時:
   - 計算當前時間與開始時間的差值
   - 減去累計暫停時間
   - 倒數: timeRemaining = initialTimeRemaining - 差值
   - 正數: elapsedTime = 差值

優點:
- 精確到毫秒
- 不受更新頻率影響
- 支援暫停/恢復
- 支援背景計時

【暫停時調整時間機制】

問題:
- 用戶在暫停時可能調整時間
- 恢復後應該使用新時間還是舊時間?

解決方案:
1. 暫停時更新 timeRemaining 和時間組件
2. 恢復時檢查用戶是否調整了時間
3. 如果調整:
   - 重置 startTime 為當前時間
   - 重置 initialTimeRemaining 為新時間
   - 清除 totalPausedTime
4. 如果未調整:
   - 累計暫停時間
   - 繼續原計時

方法:
adjustTimeComponentsWhenPaused()
- 在暫停狀態下手動調整時呼叫
- 直接更新 timeRemaining

stopDisplayTimer()
  (計時狀態保持,不停止計時)

返回前景:
  ↓
timerManager.appWillEnterForeground()
  ↓
updateTimerDisplay() (重新計算時間)
  ↓
startDisplayTimer() (重新訂閱)
  ↓
hasSynced = true

===========================================
八、整合機制
===========================================

【1. 與任務管理整合】

關聯邏輯:
- timerManager 持有 todoViewModel 參考
- 啟動計時時獲取當前任務:
  - 篩選今日時間範圍內的未完成任務
  - 選擇最早創建的任務
- 設置 subject 為任務標題
- 設置 currentTaskId

更新 focusTime:
- 計時結束時
- 將專注時間(分鐘)累加到任務的 focusTime
- 呼叫 todoViewModel.updateTask()

監聽變更:
- 監聽 .todoDataDidChange 通知
- 任務變更時重新獲取當前任務

【2. 與統計分析整合】

同步機制:
- timerManager 持有 staticViewModel 參考
- 計時結束時呼叫 syncFocusTimeToStatistics()

數據流:
1. 獲取當前任務的類別
2. 計算專注時間(分鐘)
3. 呼叫 staticViewModel.updateCategoryFocusTime()
4. 更新該類別的 focusTime

條件:
- 專注時間 >= 30 秒才同步
- 專注時間 < 1 分鐘不記錄到統計

【3. 與用戶認證整合】

用戶ID 管理:
- setCurrentUserId(_ userId:)
- 所有記錄綁定 userId

登出處理:
- 監聽 .userDidLogout 通知
- 呼叫 clearTimerState()
- 清除所有狀態和參考

【4. 與通知系統整合】

監聽:
- .userDidLogout - 用戶登出
- .todoDataDidChange - 任務變更

發送:
- 目前沒有發送通知
- 透過 EnvironmentObject 自動更新 UI

九、已知問題與限制
===========================================

【1. StaticViewModel 獲取問題】
- 之前嘗試透過 UIApplication 遞歸查找
- 改為直接注入 staticViewModel
- 解決了統計不更新的問題

【2. 時間限制】
- 最多 3 小時
- hours == 3 時強制 minutes/seconds = 0
- 無法設定更長的計時

【3. 背景限制】
- iOS 背景任務有時間限制
- 長時間背景可能被系統終止
- 使用時間差計算法減輕影響

【4. 拖動靈敏度】
- 拖動閾值固定為 80
- 某些用戶可能覺得不夠靈敏

【5. 記錄存儲限制】
- 使用 UserDefaults 存儲
- 記錄過多可能影響性能
- 沒有自動清理機制
