===========================================
用戶設定功能 (User Settings & Profile)
===========================================

【功能概述】
用戶設定功能提供個人資料管理、應用偏好設定、學習計劃設定、以及 Google OAuth 認證整合。
本功能是整個應用的基礎，負責身份驗證、用戶資料管理、應用設定同步等核心功能。

【1. UserProfile - 用戶個人資料】
檔案位置: studyAssistant/Models/UserProfile.swift

資料結構:
struct UserProfile: Codable, Identifiable {
    var id: String                  // 用戶唯一識別碼 (Firebase Auth UID)
    var email: String               // 電子郵件
    var username: String            // 使用者名稱
    var motivationalQuote: String   // 座右銘/激勵語
    var targetDate: Date            // 目標日期
    var userGoal: String            // 學習目標
    var learningStage: String       // 學習階段
    var isVIP: Bool                 // VIP 狀態
    var lastLoginAt: Date           // 最後登入時間
    var version: Int                // 版本號（用於衝突解決）
    var lastSyncedAt: Date?         // 最後同步時間
    var isDeleted: Bool             // 軟刪除標記
}

核心方法:

1. static func defaultProfile() -> UserProfile
   作用: 創建預設個人資料
   返回值:
   - username: "新用戶"
   - motivationalQuote: "開始你的學習之旅吧！"
   - targetDate: 180天後
   - learningStage: "大學"
   - isVIP: false
   - version: 1

2. init?(documentId: String, data: [String: Any])
   作用: 從 Firestore 文檔數據初始化
   特點:
   - 安全解析所有欄位
   - 處理 Timestamp 轉換
   - 可選欄位有預設值
   - 解析失敗返回 nil

3. var toFirestore: [String: Any]
   作用: 轉換為 Firestore 可儲存格式
   處理:
   - Date 轉換為 Timestamp
   - 可選值安全處理
   - 返回字典格式

用途:
- 儲存用戶基本資訊
- 提供個人化顯示（座右銘、目標日期）
- 追蹤登入狀態和活躍度
- 版本控制與衝突解決

【Authentication - Google OAuth 整合】
檔案位置: studyAssistant/Authentication.swift

核心方法:

1. @MainActor func googleOauth() async throws
   作用: 執行 Google OAuth 登入流程

   完整流程:
   ┌─────────────────────────────────────────────────────┐
   │ 第一步: 網路連接檢查                                │
   │ ├─ checkNetworkConnection()                         │
   │ │  └─ 使用 NWPathMonitor 檢查網路狀態               │
   │ │     ├─ 顯示網路類型 (WiFi/行動數據)               │
   │ │     ├─ 顯示可用網路介面                           │
   │ │     └─ 返回是否已連接                             │
   │ │                                                   │
   │ ├─ 如果標準檢測失敗:                                │
   │ │  └─ tryAlternativeNetwork()                      │
   │ │     ├─ 測試多個域名: google.com, apple.com, etc. │
   │ │     └─ canConnect(to:port:) TCP 連接測試         │
   │ │        └─ 3秒超時機制                             │
   │ │                                                   │
   │ └─ 所有測試失敗: 拋出 "無法連接到網路" 錯誤         │
   ├─────────────────────────────────────────────────────┤
   │ 第二步: Bundle ID 檢查                              │
   │ ├─ 檢查 Bundle.main.bundleIdentifier                │
   │ ├─ 預期: "eason.studyAssistant"                    │
   │ ├─ 不符時:                                          │
   │ │  ├─ DEBUG 模式: 警告但繼續                        │
   │ │  └─ RELEASE 模式: 拋出錯誤                        │
   ├─────────────────────────────────────────────────────┤
   │ 第三步: 取得 Google OAuth Client ID                │

===========================================
八、安全性與隱私
===========================================

【1. 認證安全】

Google OAuth 2.0:
- 使用官方 GoogleSignIn SDK
- 不儲存密碼
- Token 自動刷新
- 支援雙重驗證

Firebase Auth:
- ID Token 驗證
- 自動刷新機制
- 會話持久化

Bundle ID 驗證:
#if DEBUG
print("DEBUG 模式: Bundle ID 不匹配不影響功能")
#else
throw AuthenticationError.runtimeError("應用程式配置錯誤")
#endif

【2. 資料隔離】

用戶資料隔離:
- 所有資料按 userId 隔離
- Firestore Security Rules 強制執行
- Cloud Functions 驗證 Auth Token

Firestore Security Rules:
match /profiles/{userId} {
  allow read, write: if request.auth != null &&
                        request.auth.uid == userId;
}

Cloud Functions 驗證:
if (!context.auth) {
  throw new https.HttpsError('unauthenticated', '需要登入');
}
const userId = context.auth.uid;
if (data.userId !== userId) {
  throw new https.HttpsError('permission-denied', '無權限');
}

【3. 資料傳輸安全】

HTTPS 加密:
- 所有 API 請求使用 HTTPS
- Firebase 自動強制 HTTPS
- TLS 1.2+ 加密

資料驗證:
- Cloud Functions 驗證所有輸入
- 客戶端驗證基本格式
- Firestore 型別檢查

【4. 隱私保護】

最小化資料收集:
- 只收集必要的個人資訊
- 不收集敏感資訊

func loadData() async {
    async let profile = dataService.fetchUserProfile()

    let (profileResult, settingsResult) = await (try profile, try settings)

    self.userProfile = profileResult
    self.appSettings = settingsResult
}
