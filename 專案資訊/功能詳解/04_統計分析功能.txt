===========================================
統計分析功能 (Statistics & Analytics)
===========================================

【功能概述】
統計分析功能提供學習數據的可視化呈現,包括每日統計、全部時間統計、類別統計、專注時間追蹤、任務完成率分析、Token 使用量追蹤、以及自動數據同步清理機制。

二、資料模型 (Static.swift)
===========================================

【LearningStatistic 結構】

struct LearningStatistic: Identifiable, Codable {
    var id: String?                   // Firestore 文件 ID (可選)
    var userId: String                // 用戶 ID
    var category: String              // 類別名稱
    var progress: Double              // 進度 (0.0-1.0)
    var taskcount: Int                // 任務總數
    var taskcompletecount: Int        // 完成任務數
    var totalFocusTime: Int           // 總專注時間(分鐘)
    var date: Date                    // 建立日期
    var updatedAt: Date               // 更新時間
    var version: Int                  // 版本號
}

【計算屬性】
completionRate: Double
- 計算: taskcompletecount / taskcount
- 返回: 任務完成率 (0.0-1.0)

【初始化】
支援預設參數初始化:
- id: 可選(新建時為 nil)
- progress, taskcount, taskcompletecount, totalFocusTime: 預設為 0
- date, updatedAt: 預設為當前時間
- version: 預設為 1

【用途】
- 按類別追蹤學習統計
- 支援專注時間累計
- 任務數量與完成率統計
- 進度追蹤

【核心方法】

1. 數據載入
   ────────

   func fetchStatistics() async

   流程:
   1. 檢查用戶登入狀態
   2. 呼叫 Cloud Functions: fetchStatistics
   3. 解析回應數據:
      - success: Bool
      - statistics: [[String: Any]]

   4. 轉換數據:
      - convertToTimestamps() 轉換日期
      - 建立 LearningStatistic 陣列

   5. 更新 statistics 屬性
   6. 呼叫 fetchTokenUsageStats()

   錯誤處理:
   - 捕獲並記錄所有錯誤
   - 設置 errorMessage
   - 輸出詳細的錯誤資訊

   func fetchTokenUsageStats() async

   流程:
   1. 檢查用戶登入
   2. 呼叫 Cloud Functions: fetchTokenUsage
   3. 解析 tokenUsage 數據:
      - totalTokens: Int
      - modelUsage: [String: Any]
      - lastUpdated: Date

   4. 建立 TokenUsageStats
   5. 更新 tokenUsage 屬性

2. 數據保存
   ────────

   func saveStatistic(_ statistic: LearningStatistic) async -> Bool

   流程:
   1. 更新 updatedAt 為當前時間
   2. 準備數據字典
   3. convertTimestampsToStrings() 轉換日期為 ISO 8601
   4. 建立參數:
      - statistic: 轉換後的數據
      - statisticId: 如果存在

   5. 呼叫 Cloud Functions: updateStatistic
   6. 檢查回應 success
   7. 重新載入 statistics
   8. 返回成功/失敗

3. 類別操作
   ────────

   func updateCategoryFocusTime(category: String, additionalTime: Int) async
   - 查找現有統計
   - 存在: 累加 totalFocusTime
   - 不存在: 建立新統計
   - 呼叫 saveStatistic()

   func updateCategoryTaskStats(category: String, completedCount: Int, totalCount: Int) async
   - 查找現有統計
   - 更新 taskcount 和 taskcompletecount
   - 呼叫 saveStatistic()

   func updateCategoryStats(category: String, completedCount: Int, totalCount: Int, totalFocusTime: Int) async
   - 同時更新任務統計和專注時間
   - 避免多次 Cloud Functions 呼叫
   - 輸出調試訊息

   func updateCategoryProgress(category: String, progress: Double) async
   - 更新類別進度值
   - 類似上述方法

4. 查詢方法
   ────────

   func totalFocusTimeByCategory() -> [String: Int]
   - 計算每個類別的總專注時間
   - 返回字典: [類別: 分鐘數]

   func getCategoryProgress() -> [String: Double]
   - 獲取每個類別的進度
   - 返回字典: [類別: 進度值]

   func getCategoryTaskCompletionRate() -> [String: Double]
   - 計算每個類別的完成率
   - 公式: taskcompletecount / taskcount
   - 返回字典: [類別: 完成率]

   func categoryCount() -> Int
   - 返回類別數量
   - 使用 Set 去重

5. 刪除操作
   ────────

   func deleteStatistic(_ statisticId: String) async

   流程:
   1. 檢查登入
   2. 呼叫 Cloud Functions: deleteStatistic
   3. 檢查回應 success
   4. 從本地陣列移除
   5. 輸出成功訊息

6. 通知處理
   ────────

   @objc func handleTaskDeleted(_ notification: Notification)

   流程:
   1. 獲取 category (從 userInfo)
   2. 忽略空或"未分類"
   3. 檢查該類別是否還有其他任務:
      - 沒有 → 刪除統計記錄
      - 有 → 更新任務計數

   邏輯:
   - 建立 TodoViewModel 實例
   - 呼叫 tasksForCategory() 獲取剩餘任務
   - 計算完成數和總數
   - 呼叫 updateCategoryTaskStats()

【輔助方法】

convertTimestampsToStrings(_ data: [String: Any]) -> [String: Any]
- 遞迴轉換 Timestamp 為 ISO 8601 字串
- 處理嵌套字典和陣列
- 用於發送到 Cloud Functions

convertToTimestamps(_ data: [String: Any]) -> [String: Any]
- 遞迴轉換日期字串或序列化格式為 Timestamp
- 支援兩種格式:
  - {_seconds: ..., _nanoseconds: ...}
  - ISO 8601 字串
- 用於解析 Cloud Functions 回應

【數據載入方法】

1. loadData()
   ────────
   根據 timeRange 載入不同數據:
   - .day: 載入計時記錄和今日任務
   - .all: 載入統計數據和 Token 使用量

2. syncStatisticsWithTasks() async
   ────────────────────────────
   同步任務與統計數據:

   流程:
   1. 設置 isSyncing = true
   2. 獲取所有統計記錄
   3. 獲取所有任務
   4. 按類別分組計算:
      - total: 任務總數
      - completed: 已完成數
      - focusTime: 總專注時間

   5. 檢查並刪除無效類別:
      - 遍歷 statistics
      - 如果類別不在實際任務中
      - 呼叫 deleteStatistic()

   6. 更新或創建有效類別:
      - 遍歷 categoryStats
      - 呼叫 updateCategoryStats()

   7. 重新載入統計數據
   8. 設置 isSyncing = false

   重要邏輯:
   - 忽略空類別、"未分類"和重複任務
   - 確保統計數據與實際任務同步
   - 自動清理無效統計

3. loadTimerRecords()
   ────────────────
   載入今日計時記錄:

   流程:
   1. 獲取所有記錄
   2. 篩選今日記錄
   3. 計算最長專注時間
   4. 計算最早開始和最晚結束時間
   5. 計算小時分布 (hourlyDistribution)
   6. 更新狀態

4. loadTodayTasks()
   ────────────────
   載入今日任務:

   流程:
   1. 獲取今日所有任務
   2. 對每個任務:
      - 非重複任務: 使用 task.isCompleted
      - 重複任務: 獲取今日實例,使用 instance.isCompleted

   3. 組合為 [(task: String, isCompleted: Bool)]
   4. 更新 todayTasks

5. updateDateDisplay()
   ────────────────
   格式化今日日期: "M月d日 EEEE"

【計算方法】

getTotalFocusTime() -> Int

getCompletionRateValue() -> Double
- 計算: 已完成任務數 / 總任務數

getColorForHour(_ hour: Int) -> Color

- short = false: "HH:MM:SS" 或 "MM:SS"
- short = true: "Xh Ym" 或 "Ym"

六、整合機制
===========================================

【與任務管理整合】

1. 任務新增:
   - ChatViewModel.checkAndCreateStatisticsForCategory()
   - 類別不存在 → 建立新統計
   - 類別存在 → 不操作(由同步機制處理)

2. 任務刪除:
   - 發送 .taskDeleted 通知
   - StaticViewModel 監聽並處理
   - 更新或刪除統計

3. 任務完成:
   - TodoViewModel.toggleTaskCompletion()
   - 不直接更新統計
   - 由 syncStatisticsWithTasks() 同步

【與計時器整合】

計時結束時:
- TimerManager.syncFocusTimeToStatistics()
- 獲取任務類別
- updateCategoryFocusTime()
- totalFocusTime += 分鐘數

條件:
- 專注時間 >= 30秒 才同步
- 轉換為分鐘(向下取整)

【與聊天助手整合】

Token 使用追蹤:
- ChatViewModel 呼叫 GPT API
- 解析 usage 資訊
- 呼叫 Cloud Functions: updateTokenUsage
- StaticViewModel.fetchTokenUsageStats() 顯示

【自動同步機制】

何時觸發:
1. onAppear (進入統計頁面)
2. 切換時間範圍
3. 任務刪除通知

syncStatisticsWithTasks() 確保:
- 統計數據與實際任務一致
- 自動清理無效類別
- 重新計算任務數和專注時間

===========================================
七、Cloud Functions API
===========================================

【使用的函數】

1. fetchStatistics
   請求: 無參數 (userId 從 Auth 獲取)
   回應: {
     success: Bool,
     statistics: [LearningStatistic]
   }

2. updateStatistic
   請求: {
     statistic: {統計數據},
     statisticId: String? (可選)
   }
   回應: {
     success: Bool,
     statisticId: String (如果是新建)
   }

3. deleteStatistic
   請求: {
     statisticId: String
   }
   回應: {
     success: Bool
   }

4. fetchTokenUsage
   請求: 無參數
   回應: {
     success: Bool,
     tokenUsage: {
       totalTokens: Int,
       modelUsage: {
         "模型名": {
           total: Int,
           prompt: Int,
           completion: Int
         }
       },
       lastUpdated: Date
     }
   }

5. updateTokenUsage
   (由 ChatViewModel 呼叫,非 StaticViewModel)
   請求: {
     model: String,
     promptTokens: Int,
     completionTokens: Int,
     totalTokens: Int
   }
