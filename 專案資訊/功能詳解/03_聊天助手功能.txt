===========================================
聊天助手功能 (AI Chat Assistant)
===========================================

【功能概述】
聊天助手是專案中最複雜的功能模組,整合 OpenAI GPT API,透過 Function Calling 機制實現智能任務管理。支援串流回應、語音輸入、多聊天室管理、以及與任務系統的深度整合。

二、資料模型 (ChatViewModel.swift)
===========================================

【1. OpenAI API 資料結構】

A. OpenAIMessage
   ────────────
   struct OpenAIMessage: Codable {
       let role: String              // "user", "assistant", "system", "function"
       let content: String           // 訊息內容
       let name: String?             // 函數名稱 (function role)
       let tool_calls: [ToolCall]?   // 工具呼叫
       let tool_call_id: String?     // 工具呼叫ID
   }

B. Tool / ToolFunction
   ────────────────
   struct Tool: Codable {
       let type: String              // "function"
       let function: ToolFunction
   }

   struct ToolFunction: Codable {
       let name: String
       let description: String
       let parameters: Parameters
   }

C. ToolCall / ToolCallFunction
   ────────────────────────
   struct ToolCall: Codable {
       let id: String
       let type: String
       let function: ToolCallFunction
   }

   struct ToolCallFunction: Codable {
       let name: String
       let arguments: String  // JSON 字串
   }

D. OpenAIRequest
   ────────────
   struct OpenAIRequest: Codable {
       let model: String              // "gpt-4.1"
       let messages: [OpenAIMessage]
       let temperature: Float
       let stream: Bool               // 串流模式
       let tools: [Tool]?             // 可用工具
       let tool_choice: String?       // "auto", "none", "required"
       let stream_options: [String: Bool]?  // 串流選項
       let reasoning_effort: String?  // GPT-5 推理程度
   }

E. 串流回應結構
   ────────────
   struct OpenAIStreamChunk: Decodable {
       let id: String
       let choices: [OpenAIStreamChoice]
       let usage: OpenAIUsage?
   }

   struct OpenAIStreamChoice: Decodable {
       let index: Int
       let delta: OpenAIStreamDelta
       let finishReason: String?
   }

   struct OpenAIStreamDelta: Decodable {
       let role: String?
       let content: String?
       let tool_calls: [ToolCallDelta]?
   }

F. Token 使用量
   ────────────
   struct OpenAIUsage: Decodable {
       let prompt_tokens: Int
       let completion_tokens: Int
       let total_tokens: Int
   }

【2. 聊天資料結構】

A. ChatMessage
   ────────────
   struct ChatMessage: Identifiable, Codable {
       let id: UUID
       var text: String
       let isMe: Bool                 // 是否為用戶訊息

       // 待確認的任務
       var pendingTasks: [PendingTask]?
       var pendingDeleteTasks: [TodoTask]?
       var pendingUpdateTasksOriginal: [TodoTask]?
       var pendingUpdateTasksUpdated: [PendingTask]?

       // 函數執行狀態
       var isWaitingFunction: Bool
       var currentExecutingFunction: String?
       var executingFunctions: [String]?
   }

   計算屬性:
   - pendingUpdateTasks: [(original: TodoTask, updated: PendingTask)]?
     └─ 將 Original 和 Updated 陣列配對

B. ChatRoom
   ────────────
   struct ChatRoom: Identifiable, Codable {
       let id: UUID
       var name: String
       var messages: [ChatMessage]
   }

C. PendingTask
   ────────────
   待確認的任務結構:

   struct PendingTask: Identifiable, Codable {
       let id: UUID
       let title: String
       let note: String
       let category: String
       let startDate: Date
       let endDate: Date
       let isAllDay: Bool
       let isCompleted: Bool
       let color: Color
   }

   特點:
   - 與 TodoTask 結構相似
   - 用於 GPT 建立任務後,等待用戶確認
   - 確認後轉換為 TodoTask 儲存

6. end_conversation

【核心方法】

1. 聊天室管理
   ────────────

   init()
   - 從 UserDefaults 載入聊天室
   - 設定 selectedRoomIndex 為最新聊天室

   createNewChatRoom()
   - 檢查 canCreateNewChatRoom
   - 新增聊天室到 chatRooms
   - 切換到新聊天室

   deleteChatRoom(at index: Int)
   - 刪除指定索引的聊天室
   - 調整 selectedRoomIndex

   canCreateNewChatRoom -> Bool
   - 檢查當前聊天室是否為空

2. GPT 串流通訊
   ────────────

   sendMessageToGPT(messages: [ChatMessage], onToken: ((String) -> Void)?) async -> String?

   主流程:
   1. 取消之前的任務
   2. 轉換 ChatMessage 為 OpenAIMessage
   3. 添加 system message (包含讀書設定和指令)
   4. 決定 tool_choice:
      - sendToGPTCount == 0: "none" (首次不使用工具)
      - sendToGPTCount == 1: "required" (第二次強制使用工具)
      - sendToGPTCount >= 2: "auto" (自動決定)

   5. 建立 OpenAIRequest
   6. 發送請求到 proxyURL
   7. 處理串流回應:
      - 收集 content chunks
      - 收集 tool_calls
      - 呼叫 onToken 回調
      - 解析 usage 資訊

   8. 處理 [DONE]:
      - 如果有 tool_calls,執行函數
      - 將函數結果添加到 allMessages
      - 遞迴呼叫 sendMessageToGPT
      - 返回完整回應

   tool_choice 策略:
   - 首次詢問: none (讓 GPT 先詢問)
   - 第二次: required (強制使用工具)
   - 之後: auto (根據上次回應類型調整)

3. 函數執行
   ────────────

   executeGetTime() -> String
   - 格式化當前時間
   - 返回 "現在時間是：yyyy/MM/dd HH:mm:ss"

   executeGetTask() async -> String
   - 優先使用 todoViewModel.tasks (快取)
   - 如果為空,呼叫 FirebaseService.fetchTodoTasks()
   - 轉換為 JSON 格式:
     {
       "currentTime": "...",
       "existingTasks": [...]
     }
   - 返回 JSON 字串

   executeSaveTask(arguments: String) async -> String
   - 解析 JSON arguments
   - 解碼 tasks 陣列
   - 對每個任務:
     - 解析日期
     - 建立 PendingTask
     - 檢查並建立類別統計
     - 呼叫 Cloud Functions createTask
     - 累計成功/失敗數量

   - 將 pendingTasks 添加到訊息
   - 重新整理任務清單
   - 返回結果字串

   executeDeleteTask(arguments: String) async -> String
   - 解析 taskIds 陣列
   - 對每個 taskId:
     - 呼叫 todoViewModel.deleteTask()
     - 累計成功/失敗數量

   - 返回結果字串

   executeUpdateTask(arguments: String) async -> String
   - 解析 tasks 陣列
   - 對每個任務:
     - 根據 taskId 獲取原始任務
     - 建立 PendingTask (更新後)
     - 添加到 pending 列表

   - 將 pendingUpdateTasks 添加到訊息
   - 等待用戶確認
   - 返回結果字串

   executeEndConversation() -> String
   - 設置 currentFunction = "end_conversation"
   - 觸發 conversationEndedSignal
   - 返回告別訊息

4. 讀書設定整合
   ────────────

   formatStudySettings() -> String
   - 根據 studySettings 格式化字串
   - 包含:
     - 讀書日期與時間偏好
     - 科目與優先順序
     - 預設任務時長
   - 用於 system message

5. 語音識別
   ────────────

   requestSpeechRecognitionPermission() async -> Bool
   - 請求語音識別權限

   startRecording()
   - 初始化 audioEngine 和 speechRecognizer
   - 建立 recognitionRequest
   - 開始錄音並識別
   - 設置靜音計時器 (1.5秒)

   stopRecording()
   - 停止錄音
   - 停止識別任務
   - 清理資源

   handleSilence()
   - 檢查是否超過1.5秒無語音
   - 自動停止錄音

6. 統計整合
   ────────────

   checkAndCreateStatisticsForCategory(_ category: String) async
   - 檢查類別是否存在於統計中
   - 不存在則建立新的 LearningStatistic
   - 呼叫 staticViewModel.saveStatistic()

7. 輔助方法
   ────────────

   parseDate(_ dateString: String) -> Date?
   - 支援多種日期格式
   - ISO 8601
   - yyyy/MM/dd HH:mm
   - yyyy-MM-dd HH:mm

   formatDate(_ date: Date) -> String
   - 格式化為 yyyy/MM/dd HH:mm

   retryOnError<T>(_ operation: () async throws -> T) async throws -> T
   - 重試機制 (最多3次)
   - 指數退避 (1秒, 2秒)

【System Message 指令】

GPT 接收的指令:
1. 目標: 用最少提問排出具體時間表
2. 語氣: 根據 studySettings.tone
3. 讀書設定: 包含日期、時間、科目
4. 特別注意:
   - 結束對話時務必呼叫 end_conversation
   - 主動結束對話的情況
   - 講話講重點
   - 不要重複呼叫同一個 function
   - 不要在文字裡提到 function 名稱
   - 詢問後不要使用 function (除了 end_conversation)
   - 只在使用者設定的時間內安排任務
   - 每個任務持續時間為設定的讀書時間
   - 不要與原有任務重疊
   - 新增/刪除/修改前要確認現在時間和已有任務
   - 操作後要解釋做了什麼改變
   - 如需安排大量任務(一兩百個),一次安排完
   - 使用者提到要幹嘛,直接依照現在時間安排
   - 沒有指定時間就是現在
   - 不要叫使用者等待
   - 直接安排任務,不要只用文字說明

【互動邏輯】

1. 發送訊息
   ────────────
   sendMessage()
   - 檢查輸入不為空
   - 添加用戶訊息到 chatRooms
   - 清空輸入
   - 添加助手佔位訊息 (等待回應)
   - 呼叫 viewModel.sendMessageToGPT()
   - 串流更新助手訊息

2. 確認任務
   ────────────
   confirmPendingTasks(for message:)
   - 遍歷 pendingTasks
   - 將 PendingTask 轉換為 TodoTask
   - 呼叫 todoViewModel.addTask()
   - 清空訊息的 pendingTasks

   confirmDeleteTasks(for message:)
   - 遍歷 pendingDeleteTasks
   - 呼叫 todoViewModel.deleteTask()
   - 清空訊息的 pendingDeleteTasks

   confirmUpdateTasks(for message:)
   - 遍歷 pendingUpdateTasks
   - 更新原始任務的屬性
   - 呼叫 todoViewModel.updateTask()
   - 清空訊息的 pendingUpdateTasks

3. 語音輸入
   ────────────
   長按語音按鈕:
   - 開始: viewModel.startRecording()
   - 結束: viewModel.stopRecording()
   - 自動更新 inputText = viewModel.transcribedText
   - 靜音1.5秒自動停止

4. 編輯標題
   ────────────
   點擊標題:

   - 顯示 TextField
   - 自動聚焦

   提交:

5. 管理聊天室
   ────────────
   iOS 26+: Toolbar 按鈕
   iOS 25-: 側邊欄長按

   批次刪除:

   - 調整 selectedRoomIndex

- 側邊欄: .move(edge: .leading)

===========================================
六、Function Calling 機制詳解
===========================================

【OpenAI Function Calling 原理】

1. 定義工具:
   - 每個工具有名稱、描述、參數結構
   - 參數使用 JSON Schema 定義

2. 發送請求:
   - 包含 tools 陣列
   - 設置 tool_choice:
     - "none": 不使用工具
     - "auto": GPT 自動決定
     - "required": 強制使用工具

3. GPT 回應:
   - 文字回應: delta.content
   - 工具呼叫: delta.tool_calls

4. 處理工具呼叫:
   - 解析 tool_calls
   - 執行本地函數
   - 將結果添加為 function role 訊息

5. 遞迴呼叫:
   - 再次呼叫 GPT API
   - GPT 根據函數結果生成最終回應

【本專案的策略】

1. 首次發送 (sendToGPTCount == 0):
   tool_choice = "none"
   - 讓 GPT 先詢問使用者需求
   - 收集資訊

2. 第二次發送 (sendToGPTCount == 1):
   tool_choice = "required"
   - 強制 GPT 使用工具
   - 確保獲取必要資訊 (getTask, getTime)

3. 後續發送 (sendToGPTCount >= 2):
   tool_choice = "auto"
   - 根據上次回應類型調整:
     - 上次是文字 → 下次 "required"
     - 上次是函數 → 下次 "auto"

4. 結束對話:
   - GPT 呼叫 end_conversation
   - 觸發 conversationEndedSignal
   - UI 顯示告別訊息

【批次操作支援】

所有操作函數支援批次:
- saveTask: tasks: [...]
- deleteTask: taskIds: [...]
- updateTask: tasks: [...]

優點:
- 減少 API 呼叫次數
- 提升效率
- 一次處理大量任務

【確認機制】

用戶確認流程:
1. GPT 執行函數 (已在雲端操作)
2. 將結果存為 pending 狀態
3. 顯示確認卡片
4. 用戶確認後清空 pending
5. 刷新 UI

好處:
- 用戶可審查操作
- 防止誤操作
- 提供透明度

【Token 追蹤】

stream_options: ["include_usage": true]
- 啟用 token 計數
- 解析 usage 欄位:
  - prompt_tokens
  - completion_tokens
  - total_tokens
- 累計到 totalTokensUsed
- 可用於成本追蹤

===========================================
七、已知問題與限制
===========================================

【1. GPT 指令遵從問題】
- GPT 有時不遵守 "不要重複呼叫 function"
- GPT 可能在文字中提到函數名稱
- 需要更多 prompt 工程優化

【2. 大量任務處理】
- 一次新增一兩百個任務可能超時
- 需要改進批次處理邏輯

【3. 日期解析】
- 支援的日期格式有限
- 某些格式可能解析失敗

【4. 語音識別準確度】
- 依賴 iOS 原生語音識別
- 準確度受口音和環境影響
- 靜音檢測時間固定 (1.5秒)

【5. Token 成本】
- 長對話 token 消耗大
- 沒有 token 限制機制
- 需要成本控制

【6. 聊天室數量限制】
- 最多 15 個聊天室
- 超過自動刪除最舊的
- 可能導致資料遺失

【7. 訊息數量限制】
- 每個聊天室最多 300 則訊息
- 超過自動刪除最舊的
- 長對話可能遺失上下文
