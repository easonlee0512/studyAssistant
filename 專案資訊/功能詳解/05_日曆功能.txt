===========================================
日曆功能 (Calendar & Assistant)
===========================================

【功能概述】
日曆功能不僅提供月曆和週曆視圖,更重要的是整合了 AI 日曆助手,能夠自動管理任務、每日自動更新、批次處理任務操作、以及撤回功能。採用 Singleton 模式確保全局一致性。

===========================================
二、CalendarAssistantViewModel (核心邏輯)
===========================================

【設計模式】

**Singleton 模式**

class CalendarAssistantViewModel: ObservableObject {
    static let shared = CalendarAssistantViewModel()
    private init() { ... }
}

為什麼使用 Singleton?
1. 確保全局只有一個日曆助手實例
2. 避免多個實例造成的狀態不一致
3. 保證每日自動更新只執行一次
4. 在背景執行不會被中斷

【核心方法】

1. 每日自動更新
   ────────────

   func performDailyAutoUpdateIfNeeded() async

   流程:
   1. 檢查 autoUpdateEnabled
   2. 檢查 autoUpdateInput 不為空
   3. 檢查今天是否已更新: hasUpdatedToday()
   4. 避免重複觸發: isAutoUpdateInProgress
   5. 等待其他操作完成: waitForCalendarUpdateAvailability()
   6. 執行更新: startUpdate()
   7. 記錄 lastUpdateDate

   hasUpdatedToday() -> Bool
   - 比較 lastUpdateDate 與當前日期
   - 使用 Calendar.isDate(_:inSameDayAs:)

   getCurrentDate() -> Date
   - 支援測試模式 (FAKE_NOW_ISO8601)
   - 返回當前日期或測試日期

   waitForCalendarUpdateAvailability() async -> Bool
   - 等待 isUpdating, isLoading, hasLoadedInitialTasks
   - 超時設定: 30秒
   - 輪詢間隔: 0.5秒
   - 返回: 是否成功等待

2. 主要更新方法
   ────────────

   func startUpdate(userInput: String) async

   流程:
   1. 取消尚未完成的更新工作
   2. 建立新的 Task
   3. 執行 runUpdate()

   func cancelUpdate()
   - 檢查 isUpdating
   - 設置狀態為"正在取消..."
   - 取消 currentUpdateTask

   private func runUpdate(userInput: String) async
   1. 設置 isUpdating = true
   2. 清空當前任務追蹤陣列
   3. 清除上次記錄
   4. 執行 performUpdate()

   private func performUpdate(userInput: String) async

   核心邏輯:
   1. 建構 system prompt: buildSystemPrompt()
   2. 建構初始訊息: [systemMessage, userMessage]
   3. 進入對話循環 (最多15輪):
      - 建立 OpenAIRequest:
        - model: "gpt-5-mini"
        - tool_choice: "required" (強制使用工具)
        - temperature: 1.0
        - stream: false
        - reasoning_effort: "low"

      - 發送請求到 proxyURL
      - 處理回應:
        - 解析 tool_calls
        - 執行對應函數: handleToolCall()
        - 將結果添加到 messages
        - 檢查 end_conversation

      - 更新狀態顯示:
        - "已新增 X 個任務"
        - "已刪除 Y 個任務"
        - "已修改 Z 個任務"

   4. 完成後:
      - 保存記錄: saveLastUpdateStatus()
      - 更新 lastXXXTasks
      - 記錄更新時間

3. 撤回功能
   ────────

   func undoLastUpdate() async

   流程:
   1. 檢查不在更新中
   2. 檢查有可撤回的操作
   3. 設置 isUpdating = true
   4. 撤回新增的任務 (刪除):
      - 遍歷 lastAddedTasks
      - 透過標題、開始/結束時間找到任務
      - 呼叫 Cloud Functions deleteTask

   5. 撤回刪除的任務 (恢復):
      - 遍歷 lastDeletedTasks
      - 重新建立 TodoTask
      - 呼叫 Cloud Functions createTask

   6. 撤回修改的任務 (恢復原狀):
      - 遍歷 lastUpdatedTasks
      - 恢復為原始數據
      - 呼叫 Cloud Functions updateTask

   7. 重新整理任務清單
   8. 如果全部成功: 清除記錄

4. Function 執行
   ────────────

   private func handleToolCall(functionName: String, arguments: String) async -> String
   - saveTask → executeSaveTask()
   - deleteTask → executeDeleteTask()
   - updateTask → executeUpdateTask()
   - end_conversation → 返回"對話結束"

   private func executeSaveTask(arguments: String) async -> String

   流程:
   1. 解析 JSON arguments 為 SaveTasksArgs
   2. ForEach(task in args.tasks):
      - 解析日期: parseDate()
      - 建立 PendingTask
      - 檢查並建立類別統計
      - 轉換為 TodoTask
      - 準備 Firestore 資料: toFirestore
      - 轉換 Timestamp: convertTimestampsToStrings()
      - 呼叫 Cloud Functions createTask
      - 成功: addedTasks.append()
      - 更新狀態: updateCurrentStatus()

   3. 重新整理任務: todoViewModel?.forceReloadTasks()
   4. 返回結果字串

   private func executeDeleteTask(arguments: String) async -> String

   流程:
   1. 解析 JSON arguments 為 DeleteTaskArgs
   2. ForEach(taskId in args.taskIds):
      - 呼叫 Cloud Functions deleteTask
      - 從本地快取獲取任務資訊(用於顯示)
      - 成功: deletedTasks.append()
      - 更新狀態

   3. 重新整理任務
   4. 返回結果字串

   private func executeUpdateTask(arguments: String) async -> String

   流程:
   1. 解析 JSON arguments 為 UpdateTasksArgs
   2. ForEach(taskArg in args.tasks):
      - 從本地快取查找原任務
      - 找不到 → 從 Firebase 重新載入
      - 建立 PendingTask (更新後)
      - 更新 TodoTask 屬性
      - 準備 Firestore 資料
      - 呼叫 Cloud Functions updateTask
      - 成功: updatedTasks.append()
      - 更新狀態

   3. 重新整理任務
   4. 返回結果字串

5. System Prompt 建構
   ──────────────

   private func buildSystemPrompt() -> String

   包含:
   1. 角色定義: "你是一個日曆安排助手"
   2. 語氣: 從 studySettings.tone 獲取
   3. 讀書設定: formatStudySettings()
   4. 當前時間: getCurrentDate()
   5. 目標:
      - 根據需求新增、刪除或修改任務
      - 完成後立即調用 end_conversation

   6. 重要規則 (16條):
      - 沒有任務需要操作就直接結束對話
      - 不要進行不必要的更新
      - 使用常識判斷什麼是"不必要的"
      - 同一天內的輕微時間差異可接受
      - 遵循使用者的讀書時段設定
      - 不要與現有任務重疊
      - 一次處理大量任務(一兩百個)
      - 沒有指定時間表示現在
      - 遵循使用者指示,不添加額外假設
      - 使用使用者的語言回應
      - ...

   7. 現有任務列表: formatExistingTasks()
      - 從 todoViewModel 獲取
      - 轉換為 JSON 格式
      - 包含 taskId, title, category, dates 等

   formatStudySettings() -> String
   - 讀書日期和時間偏好
   - 每次讀書時長
   - 格式化為可讀文字

6. 持久化相關
   ────────────

   private func loadLastUpdateStatus()
   - 從 UserDefaults 載入 LastUpdateStatus
   - 更新 lastXXXTasks

   private func saveLastUpdateStatus()
   - 將當前更新記錄保存到 UserDefaults
   - 使用 LastUpdateStatus 結構

   private func clearLastUpdateStatus()
   - 清空所有 lastXXXTasks

7. 輔助方法
   ────────

   convertTimestampsToStrings(_ data: [String: Any]) -> [String: Any]
   - 遞迴轉換 Timestamp 為 ISO 8601 字串
   - 處理嵌套字典和陣列
   - 用於發送到 Cloud Functions

   convertToTimestamps(_ data: [String: Any]) -> [String: Any]
   - 遞迴轉換日期字串為 Timestamp
   - 支援兩種格式:
     - {_seconds: ..., _nanoseconds: ...}
     - ISO 8601 字串

   parseDate(_ dateString: String?) -> Date?
   - 支援多種日期格式
   - 優先使用 ISO8601DateFormatter
   - 備用 9 種 DateFormatter 格式
   - 輸出調試訊息

   parseColor(_ colorString: String?) -> Color?
   - 解析 "R,G,B,A" 格式字串

   updateCurrentStatus()
   - 根據 addedTasks, deletedTasks, updatedTasks 數量
   - 更新 currentStatus 文字

   checkAndCreateStatisticsForCategory(_ category: String) async
   - 檢查類別統計是否存在
   - 不存在則建立新的 LearningStatistic

8. 重試機制
   ────────

   private func retryOnError<T>(maxAttempts: 5, baseDelay: 2.0, operation: () async throws -> T) async throws -> T

   邏輯:
   1. 指數退避演算法: 2^(retry-1) * baseDelay
   2. 加入隨機抖動: +0~1秒
   3. 最大延遲: 60秒
   4. 處理 429 狀態碼:
      - 讀取 Retry-After header
      - 使用建議的等待時間
   5. 最多重試 5 次
   6. 輸出詳細的重試日誌

                }
            }
        }

    if assistantViewModel.isUpdating && !currentStatus.isEmpty {

  ├─ 成功: addedTasks.append(pendingTask)
  └─ updateCurrentStatus()
  ↓
todoViewModel.forceReloadTasks()
  ↓
返回結果 "已成功新增 X 個任務"
  ↓
添加到 messages (作為 tool role)
  ↓
繼續下一輪對話

七、每日自動更新詳解
===========================================

【觸發時機】

1. App 啟動:
   - studyAssistantApp.onAppear
   - ContentView.onAppear

2. 從背景返回:
   - .scenePhase 變化
   - active → performDailyAutoUpdateIfNeeded()

3. 日期變更:
   - 跨越午夜後首次啟動

【判斷邏輯】

hasUpdatedToday() -> Bool:
1. 獲取 lastUpdateDate
2. 比較與當前日期是否同一天
3. 使用 Calendar.isDate(_:inSameDayAs:)

支援測試模式:
- 環境變數: FAKE_NOW_ISO8601
- 可模擬不同日期進行測試

【執行保護】

1. 防止重複執行:
   - isAutoUpdateInProgress 標記

2. 等待其他操作:
   - waitForCalendarUpdateAvailability()
   - 檢查 isUpdating, isLoading, hasLoadedInitialTasks
   - 超時 30 秒取消

3. 錯誤處理:
   - updateError 記錄錯誤
   - 不影響下次執行

【記錄時間】

更新成功:
- lastUpdateDate = getCurrentDate()
- 保存到 UserDefaults

下次執行:
- 檢查 hasUpdatedToday()
- 如果是新的一天,再次執行

十一、已知問題與限制
===========================================

【1. GPT 指令遵從】
- GPT 有時不遵守"不要重複呼叫"
- 可能進行不必要的更新
- 需要更精確的 prompt 工程

【2. 任務匹配】
- 撤回功能依賴標題和時間匹配
- 如果任務被修改可能找不到

【3. 超時處理】
- 10 分鐘 timeout 可能不足
- 大量任務處理可能超時

【4. 本地快取】
- executeUpdateTask 依賴本地快取
- 快取不存在時需從 Firebase 重新載入

【5. 測試模式】
- 依賴環境變數 FAKE_NOW_ISO8601
- 正式環境要確保不設置
