===========================================
二、資料模型 (TodoTask.swift)
===========================================

【1. RepeatType 枚舉】
定義任務重複類型:
- .none: 不重複
- .daily: 每天重複
- .weekly: 每週重複
- .monthly: 每月重複

實現了 Hashable 和 Codable 協議,支援比較和序列化。

【2. TaskInstance 結構】
用於表示重複任務的單個實例:

struct TaskInstance {
    let id: String              // 實例唯一ID
    var date: Date              // 實例日期
    var isCompleted: Bool       // 完成狀態
    var parentTaskId: String    // 父任務ID
}

關鍵方法:
- init?(documentId:data:) - 從 Firestore 資料建立
- toFirestore - 轉換為 Firestore 格式

【3. TodoTask 結構】
主要任務資料結構:

struct TodoTask: Identifiable, Codable, Equatable {
    // 基本資訊
    let id: String              // Firestore 文件 ID
    var title: String           // 任務標題
    var note: String            // 任務備註
    var color: Color            // 任務顏色標記
    var category: String        // 任務類別

    // 時間設定
    var isAllDay: Bool          // 是否全天任務
    var startDate: Date         // 開始日期
    var endDate: Date           // 結束日期

    // 重複設定
    var repeatType: RepeatType  // 重複類型
    var repeatEndDate: Date?    // 重複結束日期
    var instances: [TaskInstance] // 任務實例陣列

    // 狀態與追蹤
    var isCompleted: Bool       // 完成狀態
    var focusTime: Int          // 專注時間(分鐘)
    var createdAt: Date         // 建立時間
    var userId: String          // 使用者ID
}

【重要方法】

A. shouldDisplay(on date: Date) -> Bool
   判斷任務是否應該在指定日期顯示

   邏輯:
   1. 檢查是否有該日期的已完成實例
   2. 檢查日期是否在任務起始日之後
   3. 檢查是否超過重複結束日期
   4. 根據 repeatType 判斷:
      - .daily: 每天都顯示
      - .weekly: 計算週期內的偏移量
      - .monthly: 計算月內的偏移量
      - .none: 只在 startDate~endDate 區間顯示

B. hasCompletedInstance(on date: Date) -> Bool
   檢查特定日期是否有已完成的實例

C. toFirestore / init(documentId:data:)
   Firestore 資料轉換方法

【顏色處理】
使用 SwiftUI Color 類型,存儲時轉換為 RGBA 組件:
{
    "red": CGFloat,
    "green": CGFloat,
    "blue": CGFloat,
    "opacity": CGFloat
}

【核心方法】

1. 資料載入
   ────────────
   func loadTasks() async throws
   - 檢查使用者登入狀態
   - 使用快取機制(60秒內不重複載入)
   - 防止併發載入
   - 呼叫 FirebaseService.fetchTodoTasks()

   func forceReloadTasks() async throws
   - 強制重新載入,忽略快取

2. 任務 CRUD
   ────────────
   func addTask(_ task: TodoTask) async
   - 確認使用者登入
   - 設定 userId
   - 處理重複任務的 repeatEndDate
   - 呼叫 FirebaseService.saveTodoTask()
   - 強制重新載入以顯示新任務

   func updateTask(_ task: TodoTask) async throws
   - 保存原始任務狀態(用於錯誤恢復)
   - 更新 Firestore
   - 更新本地任務列表
   - 發送 todoDataDidChange 通知
   - 失敗時恢復原始狀態

   func deleteTask(_ task: TodoTask) async throws
   - 確認使用者權限(只能刪除自己的任務)
   - 呼叫 FirebaseService.deleteTodoTask()
   - 從本地陣列移除
   - 發送 taskDeleted 通知(包含類別資訊)

3. 任務完成狀態切換
   ────────────────
   func toggleTaskCompletion(_ task: TodoTask) async
   - 只處理非重複任務
   - 立即更新本地狀態(樂觀更新)
   - 加入更新佇列
   - 異步處理更新

   func processCompletionQueue() async
   - 處理完成狀態更新佇列
   - 支援重試機制(最多3次)
   - 失敗時恢復本地狀態
   - 防止併發處理

4. 任務實例管理
   ────────────
   func toggleInstanceCompletion(_ instance: TaskInstance, in task: TodoTask) async throws
   - 立即更新本地實例狀態
   - 呼叫 FirebaseService.updateTaskInstanceCompletion()
   - 失敗時恢復狀態

   func getInstancesForDate(_ date: Date, task: TodoTask) -> [TaskInstance]
   - 獲取特定日期的任務實例

   func updateInstance/deleteInstance
   - 更新或刪除任務實例

5. 任務過濾與排序
   ────────────────
   func tasksForDate(_ date: Date) -> [TodoTask]
   - 使用 task.shouldDisplay(on:) 過濾

   func tasksForCategory(_ category: String) -> [TodoTask]
   - 按類別過濾

   func sortedTasks(by date: Date) -> [TodoTask]
   - 按開始時間排序

   func sortedTasksWithCompletionStatus(by date: Date) -> [TodoTask]
   - 未完成任務在前,已完成在後
   - 同狀態按開始時間排序

6. 表單管理
   ────────
   func initNewTaskForm(selectedDate: Date)
   - 初始化新增任務表單
   - 重置所有欄位為預設值

【通知機制】

監聽通知:
- .todoDataDidChange - 資料變更時重新載入
- .userAuthDidChange - 驗證狀態變更

func saveTask()

===========================================
九、重複任務機制詳解
===========================================

【伺服器端處理】

Cloud Functions 在建立/更新重複任務時:

1. 檢查 repeatType
2. 如果不是 .none,生成任務實例:
   - 從 startDate 開始
   - 到 repeatEndDate 結束(或預設結束日期)
   - 根據 repeatType 計算間隔
     - daily: 每天一個實例
     - weekly: 每週相同星期幾一個實例
     - monthly: 每月相同日期一個實例

3. 在 Firestore 中建立 instances 子集合:
   tasks/{userId}/userTasks/{taskId}/instances/{instanceId}

【客戶端顯示】

TodoTask.shouldDisplay(on date: Date):

1. 檢查是否有該日期的已完成實例
   - 有 → 顯示(即使超出原始日期範圍)

2. 檢查日期是否在 startDate 之後

3. 檢查是否超過 repeatEndDate

4. 根據 repeatType 判斷:

   daily:
   - 直接返回 true

   weekly:
   - 計算起始日是星期幾: startW
   - 計算跨度: span = endDate - startDate + 1 天
   - 計算目標日期是星期幾: weekday
   - 計算週內偏移: shiftInWeek = (weekday - startW) % 7
   - 判斷: shiftInWeek < span

   monthly:
   - 計算目標日期是幾號: dayOfMonth
   - 計算跨度: span
   - 計算起始日是幾號: startDay
   - 計算月內偏移: shift = (dayOfMonth - startDay) % 31
   - 判斷: shift < span

   none:
   - 只在 startDate ~ endDate 範圍內顯示

【實例管理】

每個實例獨立追蹤完成狀態:

1. 顯示時:
   - getInstancesForDate() 獲取該日期的所有實例
   - 使用 TodoItemView(task:instance:) 顯示

2. 切換完成:
   - toggleInstanceCompletion()
   - 只更新該實例的 isCompleted
   - 不影響主任務或其他實例

3. 統計計算:
   - 統計類別完成數時,計算所有實例的完成數
   - getCompletedInstances() / getIncompleteInstances()

【範例】

任務:
- startDate: 2025-11-03 (週日)
- endDate: 2025-11-05 (週二)
- repeatType: .weekly
- repeatEndDate: 2025-12-31

顯示邏輯:
- span = 3 天(日,一,二)
- startW = 0 (週日)

判斷 2025-11-10 (週日):
- weekday = 0
- shiftInWeek = (0 - 0) % 7 = 0
- 0 < 3 → 顯示

判斷 2025-11-12 (週二):
- weekday = 2
- shiftInWeek = (2 - 0) % 7 = 2
- 2 < 3 → 顯示

判斷 2025-11-13 (週三):
- weekday = 3
- shiftInWeek = (3 - 0) % 7 = 3
- 3 < 3 → 不顯示
