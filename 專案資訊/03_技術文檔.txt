一、Swift / SwiftUI 技術細節
===========================================

【開發環境】
- Xcode 版本: 15.0+
- Swift 版本: 5.9+
- iOS 最低支援: 16.0
- 目標平台: iOS

【SwiftUI 核心技術】

1. 聲明式 UI
   - 使用 SwiftUI 構建所有界面
   - 自動 UI 更新（狀態驅動）
   - 預覽支援（#Preview）

2. 狀態管理
   @State - 本地狀態
   @StateObject - ViewModel 擁有權
   @ObservedObject - ViewModel 觀察
   @EnvironmentObject - 環境注入
   @Published - 發布屬性變化

3. 生命週期
   .onAppear { }
   .onDisappear { }
   .task { }
   .onChange(of:) { }

4. 導航
   NavigationStack
   NavigationLink
   .sheet()
   .fullScreenCover()

5. 列表與集合
   List
   ForEach
   ScrollView
   LazyVStack / LazyHStack

【Combine Framework】

1. Publishers
   - NotificationCenter.Publisher
   - @Published
   - PassthroughSubject

2. Operators
   - .map()
   - .filter()
   - .sink()
   - .receive(on:)

3. Subscribers
   - .sink { }
   - .assign(to:)
   - .store(in:)

4. 使用場景
   - 異步事件處理
   - 通知訂閱
   - 資料流轉換

範例:
NotificationCenter.default.publisher(for: .userProfileDidChange)
    .receive(on: RunLoop.main)
    .sink { [weak self] _ in
        Task { await self?.loadProfile() }
    }
    .store(in: &cancellables)

【Swift Concurrency】

1. async/await
   - 異步函數標記
   - 同步式寫法
   - 結構化並發

2. Task
   - Task { await ... }
   - Task.detached { }
   - Task.sleep()

3. @MainActor
   - 主線程執行
   - UI 更新保證

4. async let
   - 並行執行
   - 提升性能

範例:
func loadData() async {
    async let profile = fetchProfile()
    async let settings = fetchSettings()

    let (p, s) = await (try profile, try settings)
}

【Swift 語言特性】

1. Protocol-Oriented Programming
   protocol DataServiceProtocol { }
   extension DataServiceProtocol { }

2. Generics
   func fetch<T: Codable>() async throws -> T

3. Result Type
   Result<Success, Failure>

4. Optional Chaining
   user?.profile?.username

5. Guard Statements
   guard let userId = Auth.auth().currentUser?.uid else { return }

6. Property Wrappers
   @Published
   @StateObject
   @EnvironmentObject

【Firebase SDK 配置】

1. 安裝方式: Swift Package Manager

init() {

【Firebase Authentication】

1. 支援的登入方式:
   - Google OAuth 2.0

2. 配置步驟:
   a. Firebase Console 啟用 Google 登入
   b. 下載 GoogleService-Info.plist
   c. 配置 URL Schemes
   d. 整合 GoogleSignIn SDK

3. 使用範例:
// 登入
let credential = GoogleAuthProvider.credential(
    withIDToken: idToken,
    accessToken: accessToken
)
try await Auth.auth().signIn(with: credential)

// 登出
try Auth.auth().signOut()
GIDSignIn.sharedInstance.signOut()

// 監聽狀態
Auth.auth().addStateDidChangeListener { auth, user in
    // 處理狀態變化
}

【Firebase Firestore】

1. 資料庫類型: NoSQL 文檔資料庫

2. 集合結構:
   /users/{userId}
   /profiles/{userId}
   /tasks/{taskId}
   /statistics/{statisticId}
   /settings/{userId}
   /studySettings/{userId}
   /chatHistories/{userId}
   /timerRecords/{recordId}

3. 查詢範例:
// 獲取單一文檔
let doc = try await db.collection("tasks").document(taskId).getDocument()

// 查詢集合
let snapshot = try await db.collection("tasks")
    .whereField("userId", isEqualTo: userId)
    .whereField("isCompleted", isEqualTo: false)
    .order(by: "createdAt", descending: true)
    .getDocuments()

// 即時監聽
db.collection("profiles").document(userId)
    .addSnapshotListener { snapshot, error in
        // 處理變化
    }

4. 安全規則:
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /tasks/{taskId} {
      allow read, write: if request.auth != null &&
                            resource.data.userId == request.auth.uid;
    }
  }
}

五、OpenAI API 整合
===========================================

【API 配置】

端點: https://api.openai.com/v1/chat/completions

認證: Bearer Token (API Key)

請求頭:
{
  "Authorization": "Bearer YOUR_API_KEY",
  "Content-Type": "application/json"
}

【使用的模型】

1. gpt-4o-mini
   用途: 聊天助手、日曆助手
   特點: 快速、經濟、支援 Function Calling
   Token 限制: 128k context

2. gpt-4o
   用途: 複雜任務（備用）
   特點: 更強大、更準確
   Token 限制: 128k context

【Function Calling 實現】

1. 定義函數列表:
let functions = [
    {
        "name": "getTask",
        "description": "獲取任務資訊",
        "parameters": {
            "type": "object",
            "properties": {
                "taskId": { "type": "string" }
            },
            "required": ["taskId"]
        }
    },
    // ... 更多函數
]

2. 發送請求:
let body = [
    "model": "gpt-4o-mini",
    "messages": messages,
    "functions": functions,
    "function_call": "auto"
]

3. 處理響應:
if let functionCall = message.function_call {
    let functionName = functionCall.name
    let arguments = parseJSON(functionCall.arguments)

    // 執行對應函數
    let result = await executeFunction(functionName, arguments)

    // 將結果返回給 GPT
    messages.append(["role": "function",
                     "name": functionName,
                     "content": result])
}

【串流響應處理】

使用 Server-Sent Events (SSE):

let request = URLRequest(url: url)
request.httpMethod = "POST"
request.setValue("text/event-stream", forHTTPHeaderField: "Accept")

let (bytes, response) = try await URLSession.shared.bytes(for: request)

for try await line in bytes.lines {
    if line.hasPrefix("data: ") {
        let data = line.dropFirst(6)
        if data == "[DONE]" { break }

        let chunk = try JSONDecoder().decode(chunk)
        let content = chunk.choices[0].delta.content

        // 更新 UI
        await MainActor.run {
            currentMessage += content
        }
    }
}
