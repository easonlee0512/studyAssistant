主要修改內容總結：

建立了新的 APIService 類別，作為與 Cloud Functions 通信的統一介面。
修改了各個 ViewModel：
移除直接的 Firestore 操作
改用 APIService 進行資料存取
將實時監聽改為定期輪詢（每30秒）

需要在 Cloud Functions 端實現的 API 端點：
/getUserProfile - 獲取用戶資料
/updateUserProfile - 更新用戶資料
/getTodoTasks - 獲取待辦事項
/saveTodoTask - 儲存待辦事項
/deleteTodoTask - 刪除待辦事項
/getAppSettings - 獲取應用設定
/updateAppSettings - 更新應用設定
/getLearningStatistics - 獲取學習統計
/getTokenUsageStats - 獲取 Token 使用統計
/getStudySettings - 獲取學習設定
/updateStudySettings - 更新學習設定

安全性考慮：
所有 API 請求都需要進行身份驗證
在 Cloud Functions 中實現資料存取權限控制
使用 HTTPS 進行安全通信
