智能重新安排規則（完善版本 - 支援連續任務鏈式遞延）
【階段一：任務分類與清理】
1. 掃描所有任務，分為四類：
   A. 已完成且已過期 → 保留原位
   B. 未完成且已過期超過七天 → 標記待刪除
   C. 未完成且已過期未超過七天 → 標記待重排
   D. 未完成且未過期 → 保留原位

2. 清理規則：
   ✓ 刪除「未完成且已過期 > 7天」的任務

【階段二：識別連續同類任務鏈】
在「待重排」任務中，識別「連續任務鏈」：

連續任務鏈定義：
- 相同標題（或類別相同且標題相似）（類別為學習判定為初始設定不計算）
- 連續多天（間隔 ≤ 1天）
- 時段相同或相近（開始時間差異 ≤ 30分鐘）

範例：
  任務A：數學（1/1 09:00-10:00）
  任務B：數學（1/2 09:00-10:00）
  任務C：數學（1/3 09:00-10:00）
  → 識別為「連續任務鏈」

【階段三：優先級計算】
對所有「待重排」任務計算優先級分數：

優先級分數 = 任務原始開始時間（越早 = 越高優先）

依優先級分數由高到低排序

【階段四：智能時間分配】

==== 4.1 連續任務鏈的特殊處理（鏈式遞延） ====

若任務屬於「連續任務鏈」，使用鏈式遞延邏輯：

步驟：
1. 找出鏈中第一個「未完成且過期<7天」的任務（稱為「待補任務」）
2. 找出該鏈中所有「未來的任務」（稱為「後續鏈任務」）

3. 鏈式遞延流程：

   A. 嘗試將「待補任務」移到今天的原時段：
      - 檢查今天 09:00-10:00 是否有空檔
      - 若有空檔 → 插入「待補任務」
      - 若有衝突 → 檢查衝突任務

   B. 若今天的原時段被佔用：
      判斷衝突任務是否為「後續鏈任務」

      ┌─ 是「後續鏈任務」：
      │   1. 將今天的任務（後續鏈任務）移到明天的原時段
      │   2. 將明天的任務移到後天
      │   3. 以此類推，直到鏈的最後一個任務
      │   4. 將「待補任務」插入今天的原時段
      │
      │   範例：
      │     昨天未完成：數學A（1/1 09:00-10:00）
      │     今天原安排：數學B（1/2 09:00-10:00）
      │     明天原安排：數學C（1/3 09:00-10:00）
      │     後天原安排：數學D（1/4 09:00-10:00）
      │
      │     → 數學B 移到 1/3 09:00-10:00
      │     → 數學C 移到 1/4 09:00-10:00
      │     → 數學D 移到 1/5 09:00-10:00
      │     → 數學A 插入 1/2 09:00-10:00
      │
      └─ 不是「後續鏈任務」：
          1. 不移動衝突任務
          2. 將「待補任務」用【一般任務邏輯】重新安排
             （見 4.2 一般任務安排邏輯）

   C. 鏈的最後一個任務的特殊處理：
      - 當鏈的最後一個任務需要順延到下一天時
      - 檢查下一天的原時段是否被佔用

      ┌─ 若被佔用：
      │   改用【一般任務邏輯】（見 4.2）
      │   - 尋找當天任何可用空檔
      │   - 若無空檔，繼續順延到下一天
      │
      └─ 若未被佔用：
          直接插入原時段

==== 4.2 一般任務的安排邏輯 ====

對於非連續任務鏈的任務，或鏈式遞延失敗的任務：

1. 時間窗口規則：
   ✓ 遵守 StudySettings 的每日可用時段
   ✓ 預設工作時段：08:00-22:00
   ✓ 避免與現有任務重疊

2. 安排策略：
   A. 從今天開始，尋找「完全適配」的空檔
      - 任務時長 ≤ 空檔時長
      - 在用戶偏好時段內
      - 優先選擇與原時段最接近的空檔

   B. 若無法完全適配：
      - 檢查是否可「拆分任務」（僅適用讀書類）
      - 例：120分鐘任務 → 拆成兩個60分鐘
      - 拆分後的任務標題加上 [1/2] [2/2] 標記

   C. 若當天無空檔：
      - 順延到下一天
      - 重複步驟 A → B

3. 特殊處理：

   運動類任務：
   ✓ 優先安排在早晨（7:00-9:00）或傍晚（18:00-20:00）
   ✓ 若過期 > 3天 → 詢問是否刪除（不直接刪除）

   全天任務：
   ✓ 不受時間窗口限制
   ✓ 檢查日期是否有其他全天任務
   ✓ 若有衝突，順延到下一天

【階段五：衝突解決】
若發現時間衝突（且不在鏈式遞延範圍內）：

優先保留規則：
1. 優先級分數高的任務（原始時間更早的）
2. 若分數相同，保留「未過期」的任務
3. 被移除的任務進入下一輪安排

【階段六：執行更新】
1. 批次更新所有任務的開始/結束時間
3. 記錄所有變更，供撤回功能使用
4. 輸出變更摘要：
   - 已刪除 X 個過期任務
   - 已重排 Y 個任務（其中 Z 個為鏈式遞延）
